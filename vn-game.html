我如何用 AI 从零做出一个游戏

这是一个 完全由 AI 辅助开发的独立游戏项目。从世界观设定、28 条分支剧情生成、Ren'Py 引擎代码编写、到两轮 AI 腔文风清洗——Cursor + DeepSeek 参与了每一个环节。但最重要的是，我建立了一套人机协作的质量控制流程：AI 负责批量生成，我负责判断、筛选、精修。

📖 世界观与剧情：AI 生成 28 条分支 + 6 结局

基于初始骨架大纲，使用 DeepSeek 批量生成剧情分支。核心挑战不是"让 AI 写出来"，而是：

1. 确保每条分支的叙事差异：不能让 28 条路径读起来像同一条。我通过精确的 Prompt 约束（不同属性增益、不同叙事语调、不同侧面视角）来制造真正的多样性。

2. 暗骰机制的工程实现：当 meta_awareness ≥ 2 时，每次判定额外暗投 d20。若命中 20，触发叙述者突破第四面墙的隐藏结局（飞升结局）。这个机制的灵感来自桌游 DM 暗骰。

3. 四文件同步维护：每次叙事修改必须同步到 script.rpy（Ren'Py 引擎）、text_runner.py（终端测试）、story_text.txt（纯文本参考）、branch_tree.html（浏览器可视化分支图）。

🪓 两轮 AI 腔文风清洗：从 697 个破折号到 17 个

AI 生成文本有两个明显的"指纹"：过度使用破折号和程式化的连接词（首先/其次/然后/最后）。我的处理方式：

第一轮清洗：修复「不是…而是…」句式滥用。统计发现四个文件合计 57 处，全部替换为感官画面或具体行动描写。

第二轮清洗：统计破折号——四个文件合计 697 个。按 3000 字 1 个的标准，砍到 17 个。同时禁止无意义精确数字（"零点二秒"、"两毫米"）和程式化连接词。

这个要求让我意识到一个问题：破折号是 AI 生成中文文本时最明显的"指纹"之一。人类写作者很少密集使用破折号，但 AI 倾向于用破折号来插入补充说明、制造戏剧性停顿。
🎨 AI 生图 Prompt 工程：角色立绘与场景设计

我为游戏建立了完整的 AIGC 素材管理体系，包含：

1. 角色立绘 Prompt 库：5 个主要角色（奈尔/米迦/塞西尔/卢西恩/卡修斯），每个角色至少 3 种表情 × 2 种姿势变体的 Prompt 模板。包含核心视觉标签、基础立绘 Prompt、表情变体 Prompt。

2. 通用风格锚定：一套正向/负向 Prompt 模板，确保所有生成素材保持统一的哥特暗黑画风（参考 Darkest Dungeon / Blasphemous / Castlevania 风格）。

3. 工具选型指南：本地部署（ComfyUI + Stability Matrix / SD + ControlNet）vs 在线服务（Midjourney / Leonardo.ai / Tensor.Art）的对比推荐。

4. 工作流程 SOP：基础图 → 表情变体 → 精修处理 → 统一归档，完整记录生成参数便于复现。

📂 查看完整 Prompt 库：prompts_fog_lamp_ai_art.md · AIGC 素材管理指南

⚙️ 系统策划：属性、检定与终局分支

系统设计核心约束：低数值复杂度，高叙事分歧。5 项属性（警觉/执念/同步/权能/元叙事感知）控制分流，隐藏变量用于暗中引导，每个结局都能回溯到 2-3 个关键行为锚点。

关键设计决策：每轮至少提供一次"短期收益 vs 长期代价"选择，9 个结局池覆盖 Dawn / Knowing / Ascending / Rust / Fracture / Ember 六种基调。

🔗 这对我做 AIGC 内容运营意味着什么

这个游戏和我的 AIGC 内容运营能力是同一条思路：

1. AI 生成 → 人工质检 → 迭代优化：和 Prompt Library 里的「AIGC 内容生产 SOP」完全一致的流程。AI 可以批量产出，但质量控制必须由人来做。

2. Prompt Engineering 实操：从剧情生成到立绘设计，每一个环节都在实战中验证了 Prompt 设计的好坏——和 Prompt Library 里的「文生图 Prompt 工程」模板互相印证。

3. 多平台分发思维：游戏本身也是一种"内容形式"。从游戏到作品集展示页，是同一条内容的"多平台适配"——和「AI 辅助多平台内容适配」Prompt 同一逻辑。
