## 🧠 AI 产品经理工作方法论

### 1. AI 需求洞察：判断「要不要用 AI」

| 步骤 | 方法 | 产出 |
|------|------|------|
| **痛点挖掘** | 用户访谈（20+ 人）+ 竞品差评阅读 + 社群观察 | 痛点清单 + 场景画像 |
| **AI 可行性评估** | 技术调研 + 成本分析 + 隐私合规评估 | AI 介入决策矩阵 |
| **需求优先级** | ICE 评分 + 用户价值/技术成本矩阵 | 功能路线图 |

**核心原则**：AI 是手段不是目的。先验证「这个问题是否值得用 AI 解决」，再考虑「用哪个模型、怎么设计 Prompt」。

### 2. AI 产品规划：设计「怎么用 AI」

| 能力 | 实践 | 案例 |
|------|------|------|
| **大模型选型** | 对比 GPT-4、DeepSeek、Whisper、OpenRouter 等，评估成本/效果/隐私 | 冷启动：DeepSeek 成本低且中文理解强 |
| **功能路线图** | 从 MVP 到进阶功能的分阶段规划 | 跨境电商：单 Agent → 多 Agent → MCP 协同 |
| **场景化需求拆解** | 将抽象 AI 能力转化为具体用户场景 | 会议舱：录音转写 → 待办提取 → 周报生成 |

### 3. Prompt 工程：定义「AI 怎么工作」

| 环节 | 工作内容 | 产出 |
|------|----------|------|
| **系统提示词设计** | 定义 AI 角色、输出格式、约束条件 | Prompt 模板库 |
| **输出效果优化** | 迭代提示词 + 调整参数 + 添加示例 | 优化后的 Prompt + 效果对比 |
| **人机协同流程** | 设计 AI 与用户的交互节点 | 交互流程图 + 降级策略 |

### 4. AI 效果评估：验证「AI 有没有用」

| 评估维度 | 方法 | 指标示例 |
|----------|------|----------|
| **功能效果** | A/B 测试 + 用户反馈 | AI 诊断满意度 80%、待办提取准确率 85% |
| **业务价值** | 前后对比 + 趋势分析 | 记录耗时缩短 60%、待办完成率提升 20% |
| **用户体验** | 可用性测试 + NPS | 系统透明度 3.2→4.6（5 分制） |

## 📊 核心 AI 项目详解

### 🐾 宠伴·健康台账｜AI 健康周报 + OCR 识别

**一句话：** 通过用户研究验证 AI 介入点，设计 OCR 识别与 AI 健康周报功能，推动记录效率提升 60%。

| 维度 | 摘要 |
|------|------|
| **AI 需求洞察** | 约 **20 位**宠物主深度访谈；发现约 **83%** 用户不愿上传接种凭证到云端 → 决策：OCR 采用**本地识别**（隐私优先）。 |
| **AI 产品规划** | **OCR 识别**：对比百度 OCR、Tesseract、本地 ML Kit，选择本地方案（成本低、隐私合规）。**AI 健康周报**：基于疫苗/驱虫/花费数据生成智能报告。 |
| **Prompt 工程** | 设计「健康周报」系统提示词：输入宠物数据 → 输出结构化分析（疫苗状态、驱虫提醒、花费趋势、健康建议）。迭代 3 版优化输出格式。 |
| **AI 效果验证** | 跟踪 **5 位**种子用户；单次记录路径耗时约 **缩短 60%**；AI 周报阅读率 **70%**，用户反馈「比自己看数据更清晰」。 |

### 🎙️ 会议·反刍舱｜Whisper 转写 + 智能待办提取

**一句话：** 针对「会后待办散了、纪要复述成本高」，设计 Whisper 转写 + AI 待办提取的最小闭环。

| 维度 | 摘要 |
|------|------|
| **AI 需求洞察** | 访谈 PM/项目协作人群：会后整理纪要耗时且漏项频发。评估 Whisper 转写成本（约 $0.006/分钟）vs 用户时间成本 → 决策：AI 介入有价值。 |
| **AI 产品规划** | **MVP**：录音 → Whisper 转写 → 结构化待办。**进阶**：待办优先级自动标注、议题-待办关联、7 日趋势分析。 |
| **Prompt 工程** | 设计「待办提取」Prompt：输入转写文本 → 输出结构化待办（任务、负责人、截止日期、优先级）。迭代 5 版降低漏提率。 |
| **AI 效果验证** | 手工抽检漏提项；上线前后对比 **7 日待办完成率**呈上升趋势；Prompt 迭代后漏提率从 25% 降至 8%。 |

### 🤖 跨境电商 AI Agent｜Multi-Agent 协同 + MCP 服务总线

**一句话：** 针对跨境购物三大痛点，设计 MCP 协议下的 Multi-Agent 协同范式，实现意图识别率 92%。

| 维度 | 摘要 |
|------|------|
| **AI 需求洞察** | 跨境电商信息密度极高：关税复杂、价差大、外文评价看不懂、物流不透明。评估：传统 Chatbot 无法处理多维度信息 → 决策：采用 Agent 主动结构化范式。 |
| **AI 产品规划** | **v1.0**：单 Agent + 嵌入式 AI 卡片。**v1.3**：引入 MCP 服务总线，多 Agent 协同（选品/翻译/物流）。**v1.6**：Agent Orchestration Matrix 可视化协同。 |
| **Prompt 工程** | 设计 Orchestrator 路由 Prompt：识别用户意图 → 调度专业 Agent → 整合结果输出。每个 Agent 有独立的系统提示词和输出格式约束。 |
| **AI 效果验证** | 意图识别率 **92%**、MCP 调用延迟 < 2s、AI 诊断满意度 **80%**；系统透明度评分从 3.2 提升到 **4.6**（5 分制）。新增 6 项电商动态功能提升产品感。 |

### ❄️ 冷启动 Cold Start｜多平台 AI 接入 + 规则引擎兜底

**一句话：** 设计「规则引擎保底 + AI 增强」的双轨架构，实现离线可用 + 个性化诊断。

| 维度 | 摘要 |
|------|------|
| **AI 需求洞察** | 「想太多而不行动」的拖延场景；评估：AI 诊断个性化价值高，但需离线可用 → 决策：规则引擎保底 + AI 增强。 |
| **AI 产品规划** | **v1.0**：规则引擎原型（关键词分类 + 模板池）。**v1.1**：接入 OpenRouter/DeepSeek/SiliconFlow。**v1.5**：创意模式（游戏化启动任务）。 |
| **Prompt 工程** | 设计「诊断 + 启动任务 + 递进计划」三段式 Prompt；创意模式 Prompt：输入拖延目标 → 输出游戏化、趣味化的启动方案。 |
| **AI 效果验证** | AI 诊断满意度 **75%**；创意模式使用率 **40%**；规则引擎离线触发率 < 5%（说明 AI 覆盖率高）。 |

### 🏠 住值·租房生命成本｜AI 租房顾问

**一句话：** 设计 AI 顾问功能，基于贪心算法结果提供个性化租房建议。

| 维度 | 摘要 |
|------|------|
| **AI 需求洞察** | 租房只看月租忽略隐性成本；评估：AI 可基于计算结果提供个性化建议 → 决策：AI 作为辅助顾问。 |
| **AI 产品规划** | **v1.3**：接入 DeepSeek V4 Flash / OpenRouter，基于预算分配结果提供租房建议。 |
| **Prompt 工程** | 设计「租房顾问」Prompt：输入预算分配结果 + 用户偏好 → 输出个性化建议（区域推荐、注意事项、省钱技巧）。 |
| **AI 效果验证** | AI 顾问使用率 **60%**；用户反馈「比自己算更直观」。 |

### ✍️ 创作舱 / 剧本舱｜AI 内容生成

**一句话：** 设计 AI 灵感生成与剧情生成功能，展示文生文 AIGC 能力。

| 维度 | 摘要 |
|------|------|
| **AI 需求洞察** | 作者资料分散、断更焦虑；评估：AI 可辅助创意生成 → 决策：AI 作为灵感助手。 |
| **AI 产品规划** | **创作舱 v2.2**：AI 灵感生成（标题、开头段落、角色设定）。**剧本舱 v2.0**：AI 剧情生成（剧情走向、角色对白、场景描述）。 |
| **Prompt 工程** | 设计「创意生成」Prompt：输入主题/风格/关键词 → 输出多方案创意内容。添加风格控制参数（严肃/轻松/悬疑）。 |
