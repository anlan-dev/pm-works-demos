## 🧱 工程思维（产品之外，还看「能不能扛、能不能改」）

我认同：**产品关注价值交付，工程还要把扩展性、可观测性、可用性等非功能指标一起设计进去**——不仅实现需求，还要预先想好失败模式、容量边界与迭代成本。几个常放在心里的例子：

- **可复用与扩展**：像「Excel/表格导出」这类能力，会优先考虑「数据源适配 + 流式写入」的样板能否在多场景复用，而不是每个页面各写一套。
- **规模与体验**：导出量级过大（例如百万级行）时，默认会想内存是否扛得住、是否要限条数/限并发；耗时过长则更适合**异步任务 + 完成通知**，而不是让用户长时间卡住前端。
- **国际化与时区**：若业务未来可能跨区，会在模型层预留 **UTC + IANA 时区** 一类扩展点，避免把展示格式和业务规则绑死在同一层。
- **数据结构与成本**：访问模式明确后，再谈稀疏矩阵是否值得压缩存储——本质是 **空间/时间/复杂度** 的显式权衡，而不是一上来就为炫技优化。

本仓库 Demo 以产品与交互验证为主，完整后端链路见可选模块 `server/`；上述原则体现在架构拆分与可选同步设计上，便于答辩或入职后与研发团队对齐同一套语言。

## 🚀 核心项目

### 🐾 宠伴·健康台账 — 养宠全周期健康档案
- **核心方案**：浏览器内 Tesseract.js OCR、按宠物管理、品牌/批号、驱虫/体重、花费与看板
- **迭代亮点（v1.8）**：
  - 数据看板与年度报告（花费趋势/疫苗覆盖率/体重变化）
  - 月历可点选某天查看当日清单；“已预约/取消预约”与提醒统计联动
  - 多宠对比表含「本月花费/超期项」、健康评分
  - PWA（manifest、Service Worker、系统通知）、多主题与三端响应式
- **用户原声**：“家里两只狗终于能分开提醒了，不会再把A狗的针记到B狗头上，这个真的救命。” —— 双狗家庭用户·Jason

### 🎙️ 会议·反刍舱 — 会议转写与待办
- **产品逻辑**：浏览器端 Whisper、可选 DeepSeek/本地 OpenAI 兼容端点；**可选** Node 后端存会议并做**服务端 AI 分析**（`server/`）
- **迭代亮点（v1.4）**：
  - 待办支持状态流转（待办/进行中/阻塞/完成）与历史回写
  - 历史会议支持关键词+任务质量筛选（含超期识别）
  - 新增会议健康评分（A/B/C）与近7天KPI卡片
  - 周报/月报支持 TOP 关键词，汇报可直接复制输出
  - UI 完成手机/平板/PC 响应式优化
- **用户原声**：“以前会后待办都散在聊天记录里，现在我每天只看这一个页面就知道谁卡住了。” —— 项目经理·杨

### ✍️ 星途·创作舱 — 网文创作管理
- **功能整合**：Markdown 大纲、角色/世界观、标题灵感库、写作目标
- **迭代亮点（v1.7）**：
  - 写作目标追踪（今日/本周字数）+ 进度条
  - 章节状态流转（草稿/修改中/完稿/待审）+ 快速筛选
  - 全文搜索（章节名/角色/标签/参考书目）+ 时间线视图
  - 创作健康评分（含等级）+ 多项目对比看板
  - 周报/月报导出（含 TOP 关键词）
  - 版本快照与回滚 + 本地点评锚点 + PWA 离线缓存
- **体验细节**：多主题切换（含墨水屏模式），UI 与交互细节统一重构
- **用户原声**：“我现在真的是先开这个再写文，大纲、人物、标题都在一起，进入状态快很多。” —— 签约作者·miki

## ☁️ 可选后端与云端同步（本仓库 `server/`）

- **一键启动**：见 `server/README.md`。默认端口 `3789`，`GET /api/health` 自检。
- **会议舱**：SQLite 存会议记录；`POST /api/analyze` 由服务端持有 DeepSeek 密钥。
- **其余 Demo**：同一服务提供 `GET/PUT /api/demo/{pet-vaccine|creation-cabin|script-review}`，前端勾选「云端备份」后将本地 JSON 快照同步到服务端 SQLite。

静态页面默认仍以浏览器本地存储为主；**后端为可选增强**，便于答辩或部署演示。
