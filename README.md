## 🎯 我用什么方式工作

1. **挖痛点**：访谈、社群观察、竞品差评 → 收敛「到底谁在什么场景痛」。  
2. **定 MVP 与不做什么**：主动砍掉非核心模块（如宠伴不做社区/商城），避免功能堆叠稀释主价值。  
3. **设成功标准**：哪些行为算「用对了产品」？提醒配置率、回访意愿、会后待办漏项等——先定义再看数或结构化反馈。  
4. **验证与迭代**：原型试用、分组对比、问卷与回访 → 决策下一版优先级。  

AI（Cursor、DeepSeek、Whisper 等）是**缩短验证周期**的手段，出现在简历里的位置应是「让我更快试假设」，而不是技能清单的主角。

## 📊 核心案例

### 🐾 宠伴·健康台账｜产品负责人

**一句话：** 通过用户研究与分组验证，解决多宠家庭「疫苗遗忘」与记录摩擦，推动核心行为指标提升。

| 维度 | 摘要 |
|------|------|
| **需求发现** | 约 **20 位**宠物主深度访谈 + 应用商店差评阅读；多宠家庭约 **70%** 表示每月至少遗忘一次接种相关节点；共性需求是「简单、免费、易上手」，而非晒宠社交。 |
| **产品决策** | 定位 **轻量健康记录册**；**砍掉社区、商城**；聚焦多宠档案、疫苗日历、花费看板、智能提醒。 **OCR**：三种方案对比后选 **本地识别**，因约 **83%** 用户不愿将接种凭证上传云端（隐私优先于成本）。 **提醒**：假设「按宠定制 + 稍后/忽略」优于单一开关；实验数据表明核心意愿指标提升（见作品集图表）。 |
| **验证与结果** | 跟踪 **5 位**种子用户；迭代「自动算有效期」「按月花费视图」等后，单次记录路径用户主观反馈耗时约 **缩短 60%**。交付 **PWA**，零服务器成本完成 idea→验证→迭代闭环。 |

### 🎙️ 会议·反刍舱｜产品负责人

**一句话：** 针对「会后待办散了、纪要复述成本高」，定义最小闭环：转写 → 结构化待办 → 状态追踪 → 周报复盘。

| 维度 | 摘要 |
|------|------|
| **需求发现** | 访谈 PM/项目协作人群：会后整理纪要、从聊天记录里捞 action 耗费大量时间，且漏项频发。 |
| **产品决策** | **不做**大而全协作套件，聚焦「这一场会议的产出可被追踪」；用 Whisper 降低转写成本，用结构化字段与状态流转让「待办在哪里」只有一个入口。 |
| **验证与结果** | 多轮 Prompt 与字段模板迭代，对手工抽检漏提项做「效果复盘」；上线前后对比中 **7 日待办完成率**呈上升趋势（作品集内趋势图与叙事一致）。可选 `server/` 满足团队私有化与落库，但不作为用户主路径绑架。 |

## 🌏 行业垂直探索

### 🤖 跨境电商 AI Agent (GlobalFUN)｜AIGC 进阶验证

**一句话：** 针对跨境购物「信息不对称、语言物流障碍、信任成本高」的三大痛点，验证 **MCP (Model Context Protocol)** 下的 Multi-Agent 交互范式。

| 维度 | 摘要 |
|------|------|
| **需求发现** | 跨境电商场景信息密度极高，用户面临关税计算复杂、国内外价差大、外文评价看不懂、物流不透明等痛点。 |
| **PM 决策** | **不做** 传统的问答式 Chatbot，而是采用 **"Agent 主动结构化"** 范式，通过 MCP 服务总线调用比价、关税、物流接口，将结果以结构化卡片形式推送，显著降低用户思考负荷。 |
| **AIGC 验证** | 实现了 **嵌入式 AI 卡片**（AI 推荐、AI 翻译）与 **全局 Agent 浮窗**（深度导购、物流/关税查询）的混合交互，展示了 Agent 对意图的自动识别与路由能力。新增 **6 项电商动态功能**（实时倒计时、社交证明滚动、拼团进度、库存递减、按钮涟漪、AI 推荐切换），提升产品感与转化体验。 |
