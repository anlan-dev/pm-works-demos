## 2. 详细能力分析

### 2.1 冷启动 — 多平台 LLM 应用

**应用场景**：拖延诊断 + 个性化启动任务生成

**AIGC 能力选型过程**：

| 平台 | 优势 | 劣势 | 适用场景 |
|------|------|------|----------|
| OpenRouter | 模型丰富、兼容性好 | 成本较高 | 需要多模型切换 |
| DeepSeek | 中文理解强、成本低 | 响应速度一般 | 中文场景首选 |
| SiliconFlow | 国内访问快、价格适中 | 模型选择有限 | 国内用户优先 |

**选型决策**：多平台支持 + 离线兜底
- 不绑定单一平台，保持灵活性
- 离线规则引擎确保基本可用（AI 失败时自动降级）

**效果回收**：
- AI 诊断满意度约 80%（种子用户反馈）
- 离线兜底触发率 < 5%（稳定性验证）
- 平均响应时间 < 3 秒（用户体验）

**关键学习**：
- 多平台策略降低了单一平台依赖风险
- 离线兜底是保障用户体验的关键设计
- Prompt 工程直接影响诊断质量

---

### 2.2 宠伴 — OCR 图像识别

**应用场景**：接种凭证照片识别，自动提取疫苗名称/日期

**技术选型**：Tesseract.js（本地 OCR）

**选型依据**：
- 隐私优先：用户访谈中 83% 拒绝云上传
- 离线可用：地铁/弱网场景下仍可使用
- 零成本：开源方案，无 API 调用费用

**效果验证**：
- 识别准确率：待补充
- 用户接受度：高（本地处理，无隐私顾虑）

**关键学习**：
- OCR 结果仅作为预填，用户确认后才落库（容错设计）
- 图像质量直接影响识别准确率（需要引导用户拍照）

---

### 2.3 会议舱 — 语音转写

**应用场景**：会议录音转文字，结构化提取待办

**技术选型**：Whisper（本地语音转写）

**选型依据**：
- 隐私优先：用户反馈「不想上传公网」
- 离线可用：无网络依赖
- 准确率：Whisper 在中文场景表现良好

**效果验证**：
- 转写准确率：待补充
- 待办提取遗漏率：通过 Prompt 迭代优化

**关键学习**：
- 语音转写仅作为内容预填，用户确认后才落库
- Prompt 迭代是提升结构化提取质量的关键

---

### 2.4 Prompt Library — Prompt 工程

**应用场景**：管理多分类 Prompt，提升 AI 使用效率

**核心能力**：
- 多级分类管理（写作/营销/编程等）
- 一键复制（Clipboard API）
- 使用统计（高频 Prompt 追踪）

**效果验证**：
- 一键复制使用率约 80%
- 搜索功能使用率约 60%

**关键学习**：
- Prompt 工程是 AIGC 应用的核心能力
- 分类管理提升了 Prompt 的复用效率
- 使用统计帮助识别高频 Prompt

## 4. 面试准备要点

### 4.1 可以重点讲述的项目

1. **冷启动** — 展示 AIGC 能力选型和多平台适配经验
2. **宠伴** — 展示用户需求挖掘和效果量化能力
3. **Prompt Library** — 展示 Prompt 工程和效率提升思维

### 4.2 准备好的回答

**Q：你如何进行 AIGC 能力选型？**
> A：我会从三个维度评估：成本、效果、稳定性。以冷启动项目为例，我对比了 OpenRouter/DeepSeek/SiliconFlow 三个平台，最终选择多平台支持 + 离线兜底策略，既保证了灵活性，又确保了基本可用性。

**Q：你如何保障 AI 功能的稳定性？**
> A：我会设计离线兜底机制。以冷启动项目为例，当 AI 调用失败时，自动降级到规则引擎，确保基本功能可用。实际运行中，离线兜底触发率 < 5%。

**Q：你如何进行效果回收？**
> A：我会通过量化指标和用户反馈两个维度。以宠伴项目为例，通过 A/B 测试验证提醒策略效果，设置提醒意愿由 26% 提升至 42%。
