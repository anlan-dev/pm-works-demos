## 2. 详细能力分析

### 2.1 冷启动 — 多平台 LLM 应用

**应用场景**：拖延诊断 + 个性化启动任务生成

**AIGC 能力选型过程**：

| 平台 | 优势 | 劣势 | 适用场景 |
|------|------|------|----------|
| OpenRouter | 模型丰富、兼容性好 | 成本较高 | 需要多模型切换 |
| DeepSeek | 中文理解强、成本低 | 响应速度一般 | 中文场景首选 |
| SiliconFlow | 国内访问快、价格适中 | 模型选择有限 | 国内用户优先 |

**选型决策**：多平台支持 + 离线兜底
- 不绑定单一平台，保持灵活性
- 离线规则引擎确保基本可用（AI 失败时自动降级）

**效果回收**：
- AI 诊断满意度约 80%（种子用户反馈）
- 离线兜底触发率 < 5%（稳定性验证）
- 平均响应时间 < 3 秒（用户体验）

**关键学习**：
- 多平台策略降低了单一平台依赖风险
- 离线兜底是保障用户体验的关键设计
- Prompt 工程直接影响诊断质量

---

### 2.2 宠伴 — OCR 图像识别

**应用场景**：接种凭证照片识别，自动提取疫苗名称/日期

**技术选型**：Tesseract.js（本地 OCR）

**选型依据**：
- 隐私优先：用户访谈中 83% 拒绝云上传
- 离线可用：地铁/弱网场景下仍可使用
- 零成本：开源方案，无 API 调用费用

**效果验证**：
- 识别准确率：待补充
- 用户接受度：高（本地处理，无隐私顾虑）

**关键学习**：
- OCR 结果仅作为预填，用户确认后才落库（容错设计）
- 图像质量直接影响识别准确率（需要引导用户拍照）

---

### 2.3 会议舱 — 语音转写

**应用场景**：会议录音转文字，结构化提取待办

**技术选型**：Whisper（本地语音转写）

**选型依据**：
- 隐私优先：用户反馈「不想上传公网」
- 离线可用：无网络依赖
- 准确率：Whisper 在中文场景表现良好

**效果验证**：
- 转写准确率：待补充
- 待办提取遗漏率：通过 Prompt 迭代优化

**关键学习**：
- 语音转写仅作为内容预填，用户确认后才落库
- Prompt 迭代是提升结构化提取质量的关键

---

### 2.4 Prompt Library — Prompt 工程

**应用场景**：管理多分类 Prompt，提升 AI 使用效率

**核心能力**：
- 多级分类管理（写作/营销/编程等）
- 一键复制（Clipboard API）
- 使用统计（高频 Prompt 追踪）

**效果验证**：
- 一键复制使用率约 80%
- 搜索功能使用率约 60%

**关键学习**：
- Prompt 工程是 AIGC 应用的核心能力
- 分类管理提升了 Prompt 的复用效率
- 使用统计帮助识别高频 Prompt

---

### 2.5 跨境电商 — 多 Agent 协同 + MCP 服务总线

**应用场景**：选品/翻译/物流多Agent协同，Orchestrator智能路由，MCP标准化通信

**AIGC 能力选型**：

| 平台 | 优势 | 劣势 | 适用场景 |
|------|------|------|----------|
| OpenRouter | 模型丰富、兼容性好 | 成本较高 | 需要多模型切换 |
| DeepSeek | 中文理解强、成本低 | 响应速度一般 | 中文场景首选 |
| SiliconFlow | 国内访问快、价格适中 | 模型选择有限 | 国内用户优先 |

**核心 AI 能力**：
- **意图识别**：Orchestrator 识别用户意图，路由到专业 Agent
- **多 Agent 协同**：选品/翻译/物流 Agent 并行工作，MCP 协议通信
- **ProgressCompiler**：长对话上下文压缩，维持状态连贯性
- **离线规则引擎**：AI 失败时自动降级到规则引擎

**效果验证**：
- AI 诊断满意度约 80%
- 多 Agent 协同响应时间 < 5 秒
- 用户对路由过程理解度提升 60%
- 系统透明度评分从 3.2 提升到 4.6（5分制）

**关键学习**：
- 多 Agent 架构让每个环节可独立迭代
- MCP 协议降低了 Agent 间耦合，新 Agent 接入 < 1 天
- 可视化路由过程是建立用户信任的关键设计
- 电商产品感（倒计时/库存递减/社交证明）是转化的核心

---

### 2.6 PixelPick — 前端模拟 AI 评分

**应用场景**：批量上传图片后自动评分和 Best 精选

**技术选型**：纯前端算法（不依赖 AI API）

**评分维度**：
| 维度 | 权重 | 评估方式 |
|------|------|----------|
| 构图 | 30% | 黄金分割/三分法检测 |
| 曝光 | 25% | 直方图分析 |
| 对焦 | 25% | 边缘锐度检测 |
| 色彩 | 20% | 饱和度/色相分布 |

**选型依据**：
- Demo 阶段零 API 成本
- 离线可用，无网络依赖
- 为未来接入真实 AI（Google Vision / Replicate）预留接口

**关键学习**：
- 前端模拟算法可快速验证产品逻辑，再决定是否接入真实 AI
- 评分算法需在产品说明中标明「模拟」，避免误导

---

### 2.7 雾灯残页 — AIGC 全流程开发

**应用场景**：从世界观到角色到对话到导出的完整文字冒险游戏开发

**AI Agent 协作模式**：

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| Agent 1 | 世界观构建 | 初始概念 | 世界设定/规则/历史 (worldbuilding.md) |
| Agent 2 | 角色设计 | 世界观 | 角色档案/关系/动机 (character_profiles.md) |
| Agent 3 | 对话生成 | 角色+场景 | 分支对话/选项/属性变化 (story_text.txt) |
| Agent 4 | 品质审查 | 全部输出 | 一致性检查/节奏分析/情感弧线 |

**效果验证**：
- 生成 4 章可玩文字冒险游戏
- 5 个 AIGC 生成资产文件（可溯源）
- PM 角色：决策（接受/拒绝/修改）+ 品控标准

**关键学习**：
- AIGC 全程开发的核心不是「AI 能做什么」，而是「PM 如何指导 AI 做出好产品」
- 文件溯源（直接链接 AIGC 生成资产）让 AI 贡献可验证
- 多 Agent 协作模式可复用到其他创意产品（游戏/小说/剧本）

## 4. 面试准备要点

### 4.1 可以重点讲述的项目

1. **冷启动** — 展示 AIGC 能力选型和多平台适配经验
2. **宠伴** — 展示用户需求挖掘和效果量化能力
3. **Prompt Library** — 展示 Prompt 工程和效率提升思维

### 4.2 准备好的回答

**Q：你如何进行 AIGC 能力选型？**
> A：我会从三个维度评估：成本、效果、稳定性。以冷启动项目为例，我对比了 OpenRouter/DeepSeek/SiliconFlow 三个平台，最终选择多平台支持 + 离线兜底策略，既保证了灵活性，又确保了基本可用性。

**Q：你如何保障 AI 功能的稳定性？**
> A：我会设计离线兜底机制。以冷启动项目为例，当 AI 调用失败时，自动降级到规则引擎，确保基本功能可用。实际运行中，离线兜底触发率 < 5%。

**Q：你如何进行效果回收？**
> A：我会通过量化指标和用户反馈两个维度。以宠伴项目为例，通过 A/B 测试验证提醒策略效果，设置提醒意愿由 26% 提升至 42%。
