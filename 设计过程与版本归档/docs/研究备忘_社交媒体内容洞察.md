# 研究备忘 · 社交媒体内容洞察

> 定位：**语言学方法向产品/内容策略迁移**，与宠伴、会议等项目并列放入作品集「研究能力」叙事。

## 1. 研究问题

同一话题在不同社群/语境下，**怎么说、为什么说、对谁说**不同；若只看高频词，易得出误导性「爆款词」列表。

## 2. 分析框架

1. **语境**：话题、互动对象、平台规范。  
2. **身份与关系**：谁在回复谁、权力距离/亲密度。  
3. **表达意图**：劝服、吐槽、求助、站队等。  
4. **语料特征**：句长、语气词、表情符号、情感词等可计算特征。

## 3. 方法与统计

- 语料清洗与标注规范（可复核）。  
- 组间差异用**统计检验**支撑「差异是否稳定」（具体检验与工作表随项目存档，面试可说明「我做了什么检验、结论如何解释」）。

## 4. 向策略的映射

- **人群化语气**：A 群与 B 群采用不同信息密度与情绪负荷。  
- **产品启示**：提醒文案、评论区运营话术、冷启动首屏文案分层。

## 5. 局限（面试主动交代）

- 语料窗口与平台政策影响可获得性。  
- **相关≠因果**：统计显著仅说明「差异稳定」，策略上线仍需小样本验证。

## 6. 与作品集其他部分的关系

- 宠伴/会议侧：验证**线上行为与主观感受**。  
- 本研究：补齐**话语与社会维度**的需求假设来源。

## 7. 可展示物（若面试官要证据）

- 词表与特征定义一页纸。  
- 分组对比表 + 检验结果截图（脱敏）。  
- 「结论 → 策略动作」对照表（见本备忘第四节）。
