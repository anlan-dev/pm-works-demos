# 需求池与反馈台账 · 结构说明

对应作品集 **饼图（9 / 5 / 3）** 与 **横向主题条图**；用于说明「数字来自台账，不是随手画的」。

## 1. 状态枚举（饼图）

| 状态代码 | 展示标签 | 含义 |
|----------|----------|------|
| `closed` | 已闭环 | 已发布或明确 Won't fix 并告知 |
| `active` | 进行中 | 已排期开发/设计中 |
| `triage` | 待评审 | 已记录，未裁决优先级 |

**当前示例合计**：9 + 5 + 3 = **17**，表示**某一统计窗口**内台账条数，不是历史全部。

## 2. 单条工单建议字段

| 字段 | 说明 |
|------|------|
| `ticket_id` | 如 FD-2025-016 |
| `title` | 短标题 |
| `project` | pet / meeting / … |
| `status` | closed / active / triage |
| `theme` | 导出备份 / OCR / 提醒灵活性 / 多主题UI / 标签系统 / 其他 |
| `source` | interview / usability / dogfood / self |
| `created_at` / `closed_at` | 日期 |
| `release` | 关联版本或 PR |

## 3. 主题条图（横向）

- 对 `status=closed`（或「已响应」定义）按 `theme` **计数**。  
- **不必**与饼图 17 条加总一致：一条工单可只打一个主主题；也可能存在「仅记录未闭环」不计入主题条图，视你定义而定。  
- 作品集示例条数：**4, 3, 4, 2, 3**（五类主题）。

## 4. 导出与脱敏

对外只导出：`ticket_id` 替换为顺序号、删除联系人信息；描述中替换具体医院/姓名。

## 5. 面试话术

「饼图是**需求池状态结构**，条图是我**按主题关掉的问题数**，回答『我是不是在按用户痛点迭代』而不是 DAU。」
