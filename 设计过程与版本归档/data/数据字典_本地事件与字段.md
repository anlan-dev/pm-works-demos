# 数据字典 · 本地 / Demo 可记录事件与字段

> 目的：向面试官说明「**若接埋点**，我会怎么命名事件、字段怎么设计」。  
> 实际 Demo 未必全部落地，以**你本地实现为准**；此处为**可复核的设计草案**。

## 通用字段（建议每条事件都带）

| 字段 | 类型 | 说明 |
|------|------|------|
| `event_id` | string | 唯一 ID（UUID 或自增） |
| `event_name` | string | 见下表枚举 |
| `ts` | ISO8601 | 客户端时间 |
| `app_build` | string | 构建号或版本号，便于按发布切片 |
| `group` | string | 实验分组：`baseline` / `treatment` / `na` |
| `session_id` | string | 可选，同一次打开会话 |

**隐私**：默认不上传原始文本；若仅本地 JSON，`user_id` 可用匿名哈希或设备随机 ID。

## 宠伴·健康台账（建议事件）

| event_name | 触发时机 | 关键属性（示例） |
|------------|----------|------------------|
| `app_open` | 冷启动/回到前台 | `cold_start` bool |
| `pet_profile_created` | 新建档案成功 | `pet_count_after` int |
| `reminder_config_started` | 进入提醒设置 | `pet_id_hash` |
| `reminder_config_saved` | 保存提醒规则 | `rule_type`, `has_snooze` |
| `health_record_added` | 新增健康记录 | `record_type`（疫苗/驱虫等） |
| `ocr_started` / `ocr_completed` | OCR 流程 | `success` bool, `duration_ms` |
| `export_clicked` | 导出触发 | `format` |

**与图表关系**：提醒「意愿」可来自量表题 + `reminder_config_saved` 完成率 + 随访口头/NPS 草稿；**合成规则**见 `合成指数计算说明_宠伴与会议.md`。

## 会议·反刍舱（建议事件）

| event_name | 触发时机 | 关键属性 |
|------------|----------|----------|
| `meeting_session_start` | 新建/打开会议 | `source` audio/paste |
| `transcribe_done` | 转写完成 | `duration_sec`, `word_count_band` |
| `todo_extracted` | 结构化待办写入 | `todo_count` |
| `todo_status_change` | 状态流变更 | `from`, `to`, `todo_id_hash` |
| `review_opened` | 打开复盘视图 | — |
| `digest_export` | 导出纪要 | `format` |

**与图表关系**：短周期「完成趋势」由状态变更序列 + 周期截止规则 rollup 成按日/按周计数，再**归一合成指数**。

## SQL / 分析粒度（口述用）

- 实验对比：`GROUP BY group, date` 数 `DISTINCT user_id` 或事件次数。  
- 留存/回访：首开日 cohort 与后续活跃日 join（口径示例见作品集 SQL 代码块）。
