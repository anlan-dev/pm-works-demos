# 验证记录 · AB 与分组观察（小样本）

> 说明：非大规模随机实验平台；记录的是 **Demo 内的假设—对照—观察**，用于支撑产品决策而非对外宣称提升百分比。

## 宠伴 · 提醒策略

| 项目 | 内容 |
|------|------|
| **问题** | 单一粗暴提醒 vs 按宠定制 + 可忽略粒度，哪个更不打扰且更愿意配置？ |
| **假设** | 「按宠 + 灵活忽略」提升配置意愿，并与回访意愿同向。 |
| **设计** | 基线 Build vs 优化 Build；用户按可得性分组（非严格随机）。 |
| **观察** | 合成示意指数：优化相对基线在**提醒配置意愿**与**短周期回访意愿**上均为正（幅度见作品集图表，量级为 Demo）。 |
| **决策** | 全量采用灵活提醒策略；用「稍后/今日忽略」作打扰护栏。 |
| **局限** | n 小、分组非完全随机，**不声称显著性 p 值**；强调方向与一致性。 |

## 会议 · 待办完成趋势

| 项目 | 内容 |
|------|------|
| **问题** | 引入状态流与反刍提醒后，短周期「待办完成」感知是否改善？ |
| **假设** | 结构化漏项减少 → 完成趋势相对基线抬升。 |
| **设计** | 以迭代节点切分时间窗；合成示意指数（基线周=100）。 |
| **观察** | 折线随 **迭代节点1～3** 呈阶梯改善（作品集图表）。 |
| **局限** | 场次与协作者数量有限；指数含主观复盘分量时需说明。 |

## 面试追问应对

- **「为什么不做双盲？」**  
  Demo 资源有限；用**固定口径 + 同向多条证据**降低拍脑袋风险，承认样本边界。  
- **「效应量多少？」**  
  报告**相对基线的指数差**，不杜撰精确到小数点后两位的业务转化率。
