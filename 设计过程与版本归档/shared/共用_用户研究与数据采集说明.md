# 共用 · 用户研究与数据采集说明

## 1. 目标

说明作品集相关项目中**需求与证据从哪里来**，满足面试对「是否拍脑袋」的追问。

## 2. 方法组合（按项目选用）

| 方法 | 用途 | 在本作品集中的应用 | 具体执行细节 |
|------|------|---------------------|--------------|
| 半结构访谈 | 挖痛点、理场景 | 宠伴：多宠提醒、记录负担；会议：会后失忆、待办散落 | 每次访谈录音（本地存储），事后转为文字摘录，提取关键原话与结论标签 |
| 可用性走查 / 任务题 | 看关键路径摩擦 | 宠伴：录入、提醒设置；会议：转写→待办→状态流 | 设计5个核心任务（见 `ux_tasks/宠伴_tasklist.md`），记录完成率、耗时、卡顿点 |
| 轻量问卷 / 随访量表 | 量表化「意愿」「耗时感受」 | 用于合成示意指数时的输入之一（非唯一） | 5点李克特量表，问卷星/腾讯问卷收集，样本量与访谈一致 |
| 行为日志（本地/Demo） | 任务完成、功能使用 | 本地存储事件；**无**大规模埋点平台 | 用 `localStorage` 记录关键操作（建档、提醒设置、记录提交），事后导出分析 |
| 语料与统计（研究项） | 语境差异、策略映射 | 社交媒体内容洞察：特征 + 检验 | 抓取公开语料，用词频、共现等方法提取特征，卡方检验验证差异 |

## 3. 样本与招募

- **宠伴**：熟人网络 + 少量公开招募；深度覆盖约 **10～12 人次**（含重复回访的可计为覆盖人次）。招募渠道：宠物社群（微信群、小红书宠物话题）、朋友推荐。筛选标准：养宠≥1只、有疫苗/驱虫记录需求。
- **会议反刍舱**：自用迭代 + **4 名协作者** 的固定跟评 + **6～8 场会议** 演练记录。协作者背景：PM、项目经理、产品运营，每周≥2场线上会议。
- **创作舱 / 冷启动 / 剧本 / Prompt**：以作者与种子用户反馈为主，偏**场景验证**。种子用户：3-5名有写作/改稿需求的朋友。

## 4. 伦理与隐私

- 默认**本地优先**、不上传原始录音/疫苗本到公网服务器（与产品隐私决策一致）。
- 展示给第三方的内容均经**脱敏**（无真实姓名、具体机构、可识别病历细节）。
- 访谈摘录在作品集中以「匿名摘要」形式呈现。
- 访谈前告知受访者：数据仅用于产品验证，不会公开原始信息，可随时退出。

## 5. 归档建议（你可自行落地）

- 访谈：日期、时长、关键原话（脱敏）、结论标签（痛点/期望/顾虑）。
- 可用性：任务列表、成功/失败、耗时备注、截图编号。
- 需求池：单号、状态（已闭环/进行中/待评审）、关联版本。
- 问卷：收集时间、样本量、各题分布、开放题关键词聚类。

**成片文档与模板**：见同目录 `用户与实验数据_总册.md`、`数据字典_本地事件与字段.md`、`合成指数计算说明_宠伴与会议.md`、`需求池与反馈台账_结构说明.md` 及 `模板_*.md`。

## 6. 面试一句话

「我没有大厂埋点资源，所以用**小样本深访 + 任务走查 + 本地日志**把问题钉死，再用**固定口径的合成指数**看迭代是不是朝假设方向动。」
