# 共用 · 指标与数据口径说明

> 适用范围：宠伴、会议反刍舱等在作品集中展示的图表与叙事。  
> 原则：**可解释、可复核、不夸大**；与「小而美 Demo、小范围试用」一致。

## 1. 示意合成指数（柱状图、折线图）

- **定义**：将多项信号按**事先写死的权重**合成为单一指数，便于在图表上比较「前后/组间」的相对变化。
- **基线**：基线阶段或基线周统一归一为 **100**，后续点表示相对基线的比例变化（非绝对业务量）。
- **宠伴柱状图**（提醒配置意愿 / 短周期回访意愿）：输入可包括——核心任务是否完成、提醒配置量表（若使用）、随访中「是否愿意继续试用」等；具体权重见个人实验笔记；图表仅表达**方向一致性与迭代取舍**，不作对外业务承诺。
- **会议折线图**（短周期待办完成趋势）：输入可包括——待办状态流变更日志、短周期复盘问卷（若使用）；与迭代节点**时间对齐**后合成指数。

## 2. 需求池结构（饼图）

- **对象**：示例窗口内在需求台账/表单中登记的条目（已闭环 / 进行中 / 待评审）。
- **当前作品集展示**：合计 **17 条**（9 + 5 + 3），用于说明**优先级结构**，不代表用户总量或 DAU。
- **数据来源**：`backlog/tickets_2025Q4.csv`，每条工单包含：单号、项目、主题、状态、优先级、关联版本、创建时间、闭环时间。

## 3. 主题响应条数（横向条形图）

- **对象**：按主题聚类的「已响应反馈」计数（导出/备份、OCR、提醒灵活性等）。
- **说明**：与饼图口径不同——同一需求可映射多主题或只记入主线；**条数之和不必等于 17**，强调「我优先闭环了哪类问题」。
- **聚类规则**：按反馈关键词提取主题，人工归类后计数，同一反馈可能涉及多个主题。

## 4. 样本规模（与作品集文案一致）

| 场景 | 规模（量级） | 说明 |
|------|--------------|------|
| 宠伴 Demo | 约 **10～12 人次** 深度参与/随访 | 熟人网络 + 少量公开招募，非线上大流量 |
| 会议反刍舱 | 约 **6～8 场会议** 演练 + **4 名协作者** 跟评 | 自用与小范围邀请，非企业并发场景 |

## 5. SQL 代码块

- 展示的是**脱敏后的口径示例**，用来说明 JOIN/GROUP BY 思路；**非**从生产库直接导出的原始查询结果。
- **实际数据存储**：本地 SQLite（会议舱）+ localStorage（宠伴等），可导出 JSON/CSV 供分析。

## 6. 数据采集时间窗口

| 项目 | 数据采集时间 | 说明 |
|------|--------------|------|
| 宠伴 | 2025年9月-11月 | 访谈+走查+随访，覆盖MVP到迭代A/B |
| 会议舱 | 2025年11月至今 | 持续自用+协作者跟评，数据在积累中 |
| 其他Demo | 2025年10月-12月 | 场景验证为主，数据量较小 |

## 7. 面试一句话

「指标是给自己迭代用的：小样本先把**口径写死**，再看**相对变化是否和假设同向**；不做经不起追问的绝对值包装。」
