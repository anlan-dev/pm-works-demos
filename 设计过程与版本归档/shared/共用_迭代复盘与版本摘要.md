# 共用 · 迭代复盘与版本摘要

> 与线上 `index.html` 中各项目「迭代历史」一致；此处为**摘要级**列表，便于面试口述「发了什么版、为什么在此时发」。

## 通用复盘结构（五段）

1. **背景与目标**：本轮要解决的具体矛盾（一句话）。  
2. **假设**：改了什么会带动哪类行为/感受。  
3. **动作**：发布了哪些功能/文案/流程。  
4. **观察**：示意指数、反馈结构、主观耗时等是否同向变化。  
5. **下一步**：保留/回滚/加码；记入需求池。

## 宠伴·健康台账（PWA）

| 阶段 | 主题 | 要点 | 验证方式 |
|------|------|------|----------|
| MVP | 多宠档案 + 疫苗日历 + 提醒 | 砍掉非核心，验证「提醒+记录」主线 | 访谈20人，70%表示有遗忘痛点 |
| 隐私与信任 | 本地 OCR 路径 | 降低上传顾虑，先跑通提醒策略 | 走查中83%用户不愿上传云端 |
| 反馈驱动 | 自动算有效期、按月聚合花费 | 减少「算日期」「看不清月花费」摩擦 | 任务走查T3完成率100%，主观耗时缩短60% |
| 体验抛光 | 暗色主题、响应式适配 | 提升多端使用体验 | 种子用户回访意愿提升 |

## 会议·反刍舱

| 阶段 | 主题 | 要点 | 验证方式 |
|------|------|------|----------|
| MVP | 转写 → 待办 → 状态流 | 拒绝大而全协作台 | 4名协作者跟评，6-8场会议演练 |
| 结构化质量 | Prompt / 字段模板 / 漏项抽检 | 减少漏提、提升可执行性 | 漏项率从约30%降至约10% |
| 节拍 | 与短周期复盘问卷对齐 | 趋势与迭代节点时间对齐 | 待办完成率示意指数从100升至118 |

## 跨境电商 · AI Agent

| 阶段 | 主题 | 要点 | 验证方式 |
|------|------|------|----------|
| MVP | 选品助手 + 基础聊天 | 验证「选品+AI」主线，零服务器成本 | 种子用户试用，响应时间<2s |
| AI Agent | 多平台接入 + 离线兜底 | OpenRouter/DeepSeek/SiliconFlow，离线规则引擎保底 | AI诊断满意度80%，离线触发<5% |
| 多Agent协同 | 选品/翻译/物流分工 | Orchestrator路由 + MCP服务总线 | 协同响应<5s，满意度+30% |
| 可视化 | 路由面板 + Agent Matrix | 意图识别过程透明化，多Agent并行状态实时展示 | 理解度+80%，信任度3.5→4.6 |
| 产品感抛光 | UI优化 + 信任体系 + 社交证明 | 微交互动画、物流追踪、多平台比价、历史价格曲线 | 信任度+25%，预期转化+15-20% |
| 交互修复+动态功能 | IIFE修复 + 6项动态功能 + UI迁移 | 修复3个严重交互bug，新增倒计时/动态流/进度条/库存/涟漪/AI推荐 | 全部按钮响应正常，Node.js语法检查通过 |

**关键学习**：
- 多Agent架构让每个环节可独立迭代，MCP协议降低了Agent间耦合
- 动态功能（倒计时/库存递减/社交证明）是电商产品的"产品感"核心
- IIFE作用域管理是前端Demo的关键技术细节，提前关闭=全局崩溃

## 冷启动 / 创作舱 / 剧本复盘舱 / Prompt Library

- **冷启动**：规则引擎保底 + AI 增强，先保「能完成第一步」再谈个性化。验证：3-5名拖延症用户试用，任务启动率提升。
- **创作舱**：模块随种子反馈调整，强调**开写到复盘**闭环。验证：种子用户反馈"资料不再分散"。
- **剧本复盘舱**：进度快照 + 标签 + 导出，服务「改稿有据可查」。验证：热力图功能让写作习惯可视化。
- **Prompt Library**：模板化、可复制、随项目复用。验证：从散落指令变为可检索资产。
