# 剧本·复盘舱 — 迭代日志

> 记录每个版本的关键决策、用户反馈与验证结果。

---

## v1.0 (2025.12) — MVP 上线

**决策依据**：长剧本改到中后期，经常忘了某场戏为什么删。

**核心改动**：
- 剧本分场管理
- 进度快照（自动记录）
- 复盘笔记（手动填写）
- 标签系统
- 本地存储（零服务器）

**验证结果**：
- 修改回溯命中率持续改善
- 用户反馈「终于能找回当时为什么删这场戏了」

---

## v1.1 (2026.01) — 标签增强 + 导出

**决策依据**：用户需要按标签筛选和导出剧本。

**核心改动**：
- 高频标签推荐
- 历史标签统计
- 导出 Word/CSV/JSON

**验证结果**：导出功能使用率约 50%。

## v1.2 (2026.02) — 热力图 + 场景看板

**决策依据**：用户需要可视化写作进度。

**核心改动**：
- 写作热力图（streak 连续天数）
- 场景进度看板（按幕/高潮分列）

**验证结果**：热力图激励效果明显。

---

## v1.3 (2026.03) — AI 剧情生成

**决策依据**：走查发现用户「写到中后期剧情卡壳」，需要创意内容生成能力。

**AIGC 能力选型**：

| 能力 | 应用场景 | 技术方案 |
|------|----------|----------|
| 剧情走向生成 | 根据剧本设定生成 3 种剧情走向 | LLM + 剧情 Prompt |
| 角色对白生成 | 根据角色性格生成对白 | LLM + 角色 Prompt |
| 场景描述生成 | 根据场景设定生成环境描写 | LLM + 场景 Prompt |

**核心改动**：
- AI 剧情引擎（基于 LLM 的创意内容生成）
- 剧情走向生成（输入剧本设定 → 输出 3 种剧情走向）
- 角色对白生成（输入角色性格 → 输出对白）
- 场景描述生成（输入场景设定 → 输出环境描写）

**效果回收**：
- AI 剧情使用率约 50%（种子用户反馈）
- 剧情走向采纳率约 35%
- 角色对白采纳率约 25%

**关键学习**：
- 创意内容生成需要「上下文理解」能力
- 用户更喜欢「可编辑的预填」而非「直接使用」
- Prompt 工程直接影响生成质量

## v1.4 (2026.06) — UI 风格迁移 + 按钮交互排查

**决策依据**：全项目UI审查，统一迁移至标准设计系统。

**核心改动**：
- 从 Neumorphic 迁移至白卡片（#ffffff）+ 浅灰画布（#f5f5f7）
- 统一字体/主色/阴影/圆角
- 按钮交互排查，确认无IIFE/clipboard同类bug
- 合规声明模块已部署（隐私声明+本地存储）

**验证结果**：语法检查通过，按钮事件绑定检查通过。
