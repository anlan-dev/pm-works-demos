# 星途·创作舱 — 迭代日志

> 记录每个版本的关键决策、用户反馈与验证结果。

---

## v1.0 (2025.12) — MVP 上线

**决策依据**：创作者访谈发现，标题卡文和多书并行易弃坑是两大高频痛点。

**核心改动**：
- 作品管理（多书并行）
- 沉浸式写作桌面
- 标题灵感库
- 自动快照（30 秒）
- 本地存储（零服务器）

**验证结果**：
- 种子用户试用，反馈「写作体验流畅」
- 多书并行功能使用率约 40%

---

## v1.1 (2026.01) — 灵感工具 + 字数目标

**决策依据**：走查发现用户「写到一半没灵感」和「不知道写了多少」。

**核心改动**：
- 随机抽题功能
- 字数目标 + 进度条
- 写作 streak 计数

**验证结果**：日均写作字数提升约 30%。

---

## v1.2 (2026.02) — 多主题 + 数据看板

**决策依据**：用户反馈「长时间写作眼睛累」和「想看写作数据」。

**核心改动**：
- 多主题切换（默认/暗色/墨水屏/护眼）
- 数据看板（字数统计、streak、进度可视化）
- 导出 JSON

**验证结果**：暗色模式使用率约 50%。

---

## v1.3 (2026.03) — AI 灵感生成

**决策依据**：走查发现用户「写到一半没灵感」，需要创意内容生成能力。

**AIGC 能力选型**：

| 能力 | 应用场景 | 技术方案 |
|------|----------|----------|
| 文生文（标题） | 根据关键词生成 10 个标题灵感 | LLM + Prompt 工程 |
| 文生文（开头） | 根据标题生成 3 个开头段落 | LLM + 风格控制 |
| 文生文（角色） | 根据设定生成角色背景故事 | LLM + 角色 Prompt |

**核心改动**：
- AI 灵感引擎（基于 LLM 的创意内容生成）
- 标题生成（输入关键词 → 输出 10 个标题）
- 开头段落生成（输入标题 → 输出 3 个风格不同的开头）
- 角色设定生成（输入基本设定 → 输出角色背景故事）

**效果回收**：
- AI 灵感使用率约 60%（种子用户反馈）
- 标题采纳率约 40%（用户选择使用 AI 生成的标题）
- 开头段落采纳率约 30%

**关键学习**：
- Prompt 工程直接影响生成质量（需要多轮迭代）
- 用户更喜欢「可编辑的预填」而非「直接使用」
- 创意内容生成需要「风格控制」能力
