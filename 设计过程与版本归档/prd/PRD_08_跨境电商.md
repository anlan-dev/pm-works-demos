## 1. 背景

跨境电商场景信息密度极高，用户面临三大核心痛点：
1. **信息不对称**: 关税计算复杂、国内外价格差异大、真假难辨。
2. **语言与物流障碍**: 外文评价看不懂、国际物流时效不透明、清关流程繁琐。
3. **信任成本高**: 缺乏实时的正品验证与跨境售后保障。

传统 AI 助手多为问答式（Q&A），但在高密度场景下用户往往不知道该问什么，导致决策链路长、转化率低。

## 2. 目标

- **降低认知门槛**: AI Agent 主动结构化展示关键信息（关税、比价、物流）。
- **缩短决策路径**: MCP 实现一站式服务闭环。
- **提升转化率**: 多 Agent 协同（选品、翻译、物流）提供个性化跨境体验。
- **社交裂变**: 阶梯拼团 + 实时动态 + 限时倒计时驱动分享转化。

## 3. 非目标

- 替代大型跨境电商平台（天猫国际、亚马逊）。
- 构建真实的支付与清关系统（Demo 阶段为模拟）。
- 支持所有小语种实时翻译（聚焦中日英）。

## 8. 核心 PM 决策

### 决策 1：Agent 视角 —— 为什么是"主动结构化"？
- **背景**: 跨境场景信息密度极高，用户不知道该问什么。
- **决策**: Agent 主动结构化展示关键信息，而非等待用户提问。
- **逻辑**: 通过 MCP 调用比价/关税/物流服务，将结果以结构化卡片推送，减少用户"思考负荷"。

### 决策 2：MCP 架构 —— 为什么用服务总线？
- **背景**: 跨境业务涉及多个外部服务，直接耦合极其脆弱。
- **决策**: 引入 MCP 作为 AI Agent 的服务总线。
- **逻辑**: Agent 只需发送 Intent，MCP 负责路由到具体服务。微服务化架构具备极强扩展性。

### 决策 3：交互形式 —— 为什么是"嵌入式 + 浮窗"？
- **背景**: 纯对话机器人容易让用户迷失，纯列表页缺乏智能感。
- **决策**: 嵌入式 Agent 卡片 + 全局 Agent 浮窗。
- **逻辑**: 嵌入式用于高频低门槛信息（推荐/翻译），浮窗用于深度长尾需求（关税计算/物流查询）。

### 决策 4：社交裂变 —— 为什么用阶梯拼团？
- **背景**: 跨境商品单价高，用户决策周期长。
- **决策**: 3档阶梯拼团（3/5/10人），人越多越便宜。
- **逻辑**: 利用社交关系链降低获客成本，实时动态+FOMO心理驱动分享。

### 决策 5：兼容性 —— 为什么用 document.execCommand 兜底？
- **背景**: navigator.clipboard API 在 file:// 协议下不可用。
- **决策**: 分享/复制功能使用 document.execCommand('copy') + 临时 textarea 兜底。
- **逻辑**: 确保 Demo 在任何打开方式下都能正常工作。
