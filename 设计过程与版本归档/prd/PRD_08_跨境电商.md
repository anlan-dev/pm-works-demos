## 1. 背景

跨境电商场景信息密度极高，用户面临三大核心痛点：
1. **信息不对称**: 关税计算复杂、国内外价格差异大、真假难辨。
2. **语言与物流障碍**: 外文评价看不懂、国际物流时效不透明、清关流程繁琐。
3. **信任成本高**: 缺乏实时的正品验证与跨境售后保障。

传统的 AI 助手多为问答式（Q&A），但在这种高密度场景下，用户往往不知道该问什么，导致决策链路长、转化率低。

## 2. 目标

- **降低认知门槛**: 通过 AI Agent 主动结构化展示关键信息（关税、比价、物流）。
- **缩短决策路径**: 利用 MCP (Model Context Protocol) 实现一站式服务闭环。
- **提升转化率**: 通过多 Agent 协同（选品、翻译、物流）提供个性化跨境体验。

## 3. 非目标

- 替代大型跨境电商平台（如天猫国际、亚马逊）。
- 构建真实的支付与清关系统（Demo 阶段为模拟）。
- 支持所有小语种的实时翻译（聚焦中日英常用语言）。

## 4. 核心用户故事

1. As 跨境购物新手，我要输入模糊需求就得到**结构化的选品推荐**，so that 不用自己费力比价。
2. As 价格敏感型用户，我要看到**关税与汇率的实时计算**，so that 清楚知道到手价。
3. As 语言障碍用户，我要**一键翻译外文评价**，so that 了解商品真实口碑。
4. As 物流焦虑用户，我要查询**国际物流时效与清关状态**，so that 安心等待收货。

| 层级 | 内容 |

| 保底 | 跨境商品展示、多语言切换、汇率与关税计算 |

## 6. 核心 PM 决策

### 决策 1：Agent 视角的选择 —— 为什么是“主动结构化”？
- **背景**: 跨境场景信息密度极高，用户往往不知道该问什么。
- **决策**: 采用 **“Agent 主动结构化”** 范式。
- **逻辑**: Agent 在后台通过 MCP 调用 `比价服务`、`关税库`、`物流接口`，将结果以 **结构化卡片** 的形式直接推送到 UI 层，显著减少用户的“思考负荷”。

### 决策 2：MCP 架构选型 —— 为什么要用服务总线？
- **背景**: 跨境业务涉及多个外部服务（支付、物流、翻译、汇率），直接耦合会导致系统极其脆弱。
- **决策**: 引入 **MCP (Model Context Protocol)** 作为 AI Agent 的服务总线。
- **逻辑**: Agent 只需向 MCP 发送意图（Intent），MCP 负责路由到具体的 `商品服务`、`翻译服务` 或 `物流服务`。这种 **微服务化** 的架构使得 Demo 具备了极强的扩展性。

### 决策 3：交互形式的权衡 —— 为什么是“嵌入式 + 浮窗”？
- **背景**: 纯对话机器人容易让用户迷失，纯列表页又缺乏智能感。
- **决策**: 采用 **“嵌入式 Agent 卡片 + 全局 Agent 浮窗”** 的混合交互。
- **逻辑**: 
  - **嵌入式**: 用于高频、低门槛的信息展示（如“AI 推荐”、“AI 翻译评价”）。
  - **浮窗**: 用于深度、长尾的需求（如“帮我算算这批货的总关税”），作为专家级助手的入口。

## 7. 指标（Demo）

- **决策效率**: 用户从“看到商品”到“点击支付”的平均时长（预期缩短 20%）。
- **Agent 触点率**: 用户主动点击“AI 推荐”或“AI 翻译”的比例。
- **MCP 调用成功率**: 监控 Agent 通过 MCP 获取服务的成功率（Demo 中模拟为 100%）。

## 8. 风险与合规

- **数据隐私**: 跨境购物涉及用户收货地址与支付信息，需符合 GDPR 与各地数据出境法规。
- **关税政策变动**: 各国关税政策频繁调整，需建立关税库的动态更新机制。
- **物流丢包风险**: 国际物流链路长，需明确赔付责任与保险机制。
