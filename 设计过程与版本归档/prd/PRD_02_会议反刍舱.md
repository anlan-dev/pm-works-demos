## 1. 背景与问题

会后**纪要复述成本高**，待办落在聊天与备忘录里**不可追踪**，导致「会上说过、会后没落地」反复出现。

## 2. 目标与非目标

**目标**

- 最小闭环：**转写 → 结构化待办 → 状态流 → 短周期复盘**。  
- 可感知成功：**整理耗时下降**、**待办完成趋势改善**（Demo 用示意指数）。

**非目标**

- 替代完整企业协作套件（权限、OKR、全量项目管理）。  
- 默认强制云上传（与隐私敏感场景冲突）。

## 3. 用户故事（节选）

1. As 会议组织者，我要录音转文字并抽待办，so that 不必全程手写。  
2. As 执行人，我要待办有状态与截止，so that 可追踪。  
3. As 敏感场景用户，我要本地处理路径，so that 不外传原始音频。

## 4. 功能范围

| 模块 | 说明 |
|------|------|
| 转写 | 本地 Whisper 路径 +（可选）后续结构化 |
| 待办 | 字段模板、漏项抽检、优先级 |
| 状态流 | 驱动「短周期完成」观测 |
| 复盘 | 与迭代节点对齐的趋势回顾 |

## 5. 技术取舍（产品视角）

- **先做小闭环**：避免「功能很多、主线跑不通」。  
- **Prompt/模板**持续迭代：结构化质量是可验收的。  
- 私有化部署可作为**增强**，不绑架默认体验。

## 6. 指标与验证

- 输入：待办创建量、状态变更、复盘问卷（若启用）。  
- 输出：短周期待办完成**示意指数**（基线周=100）；详见共用指标文档。  
- **样本**：约 **6～8 场会议** + **4 名协作者**跟评。

## 7. 风险

- 转写错误传导至待办质量 → 需抽检与人工快速修正入口。  
- 不同会议类型字段差异 → 模板需可配置/可扩展。
