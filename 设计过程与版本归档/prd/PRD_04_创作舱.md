# PRD · 创作舱（沉浸式创作桌面）

| 字段 | 内容 |
|------|------|
| **文档版本** | v0.2（过程归档） |
| **负责人** | 王天娇 |
| **状态** | Demo 可用 |

## 1. 背景

网文/剧本作者资料散在 Word、备忘录与群聊，写作时需**反复切换窗口**，心流打断。

## 2. 目标

- 「打开就能写」：角色、大纲、正文在同一叙事空间可触达。  
- **首次打开到开始写作**路径短（Demo 用任务计时走查）。

## 3. 用户故事（节选）

1. As 作者，我要角色设定与正文同屏可达，so that 不写错人设。  
2. As 卡文作者，我要标题/灵感库抽选，so that 快速起笔。  
3. As 夜间用户，我要暗色/护眼主题，so that 长时间可写。

## 4. 功能范围

- Markdown 编辑、角色设定面板、大纲折叠、多主题。  
- 种子用户反馈驱动的模块排序与默认布局优化。

## 5. 非目标

- 完整多人实时协作（本阶段）。  
- 替代专业排版/出版流程。

## 6. 指标（Demo）

- 核心任务完成率、模块点击率、主观「找设定耗时」量表（若有）。  
- 反馈条目进入需求池，支撑排序。

## 7. 风险

- 功能增多导致「又变回 IDE」→ 坚持**单场景心流**优先。
