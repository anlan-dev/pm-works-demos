# PRD · 冷启动（拖延冷启动助手）

| 字段 | 内容 |
|------|------|
| **文档版本** | v0.2（过程归档） |
| **负责人** | 王天娇 |
| **状态** | Demo 可用 |

## 1. 背景

用户面对拖延目标时常「想太多不行动」——需要**可立即执行的第一步**，而非鸡汤长文。

## 2. 目标

- 输入一个拖延目标 → 输出 **诊断 + 启动任务 + 后续计划** 三段式结构。  
- **离线可完成核心路径**（规则引擎保底）；在线时可用 **AI 多平台** 增强个性化。

## 3. 非目标

- 替代心理咨询或医疗诊断。  
- 做成复杂项目管理工具。

## 4. 核心用户故事

1. As 拖延用户，我要输入模糊目标就得到**具体第一步**，so that 不用自己想怎么开始。  
2. As 离线用户，我要断网仍可用，so that 地铁/弱网场景可启动。  
3. As 进阶用户，我要接 AI，so that 输出更贴合语境。

## 5. 范围

| 层级 | 内容 |
|------|------|
| 保底 | 规则引擎生成可执行第一步 |
| 增强 | AI 诊断与计划润色（可用多模型接入） |
| 体验 | PWA 就绪、轻量界面 |

## 6. 指标（Demo）

- **首次会话内完成「启动任务」勾选/确认**（行为代理）。  
- 定性：输出是否「可执行、可复述」（走查表）。

## 7. 风险与合规

- AI 输出需二次确认，避免「错误但听起来专业」的建议；敏感领域加免责声明。
