## 1. 背景

在多个 Demo 迭代中重复写相似 Prompt，**重复劳动多**、版本分散；需要**可分类、可复制、可演进**的库。

## 2. 目标

- 按场景分类（PRD 提炼、用户故事、数据口径说明、会议纪要结构化等）。  
- **一键复制**到剪贴板，对接 Cursor / 其他模型工具的工作流。  
- 随项目沉淀「好用/不好用」注记（轻量，不必做复杂评测平台）。

## 3. 用户故事

1. As 产品实习生，我要找到「写用户故事」模板，so that 输出结构一致。  
2. As 独立开发者，我要复制带约束的 System Prompt，so that 减少幻觉格式。  
3. As 维护者，我要给条目打标签，so that 检索快。

## 4. 功能范围

- 分类树 / 标签、搜索、复制按钮、（可选）最近使用。  
- 与 `index.html`、各 HTML Demo 交叉引用说明。

## 5. 非目标

- 替代企业级 Prompt 评测与流量分摊系统。

## 6. 指标（Demo）

- 复制次数 / 活跃条目（若有本地统计）；否则以**维护条目数 + 自用频次**定性。  
- 版本更新是否与项目迭代同步（文档级检查）。

## 7. 风险

- 模型升级导致旧 Prompt 失效 → 条目注明**模型族/日期**与**回退策略**。
