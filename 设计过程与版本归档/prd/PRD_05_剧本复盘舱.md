# PRD · 剧本复盘舱

|------|------|

## 1. 背景

长剧本修改到中后期，常忘记**某场戏为何删、某角色动机为何改**——缺少可复盘、可导出的修改痕迹。

## 2. 目标

构建 **剧本 – 进度 – 复盘** 闭环：每次修改可追溯到「当时为什么」。

## 3. 用户故事（节选）

1. As 编剧，我要修改自动记快照，so that 可回溯删戏原因。  
2. As 长篇作者，我要按标签筛历史，so that 快速定位某类改动。  
3. As 交稿方，我要导出 Word/CSV/JSON，so that 对接导演/制片格式。

## 4. 功能范围

| 功能 | 说明 |
|------|------|
| 进度快照 | 与场次/场记绑定 |
| 标签 | 删戏/改台词/结构等分类 |
| 热力图/看板 | 视觉化管理写作密度与修改节奏 |
| 导出 | 多格式交付 |

## 5. 指标（Demo）

- **回溯命中率**：问「某段为什么改」能否 1 分钟内定位记录（走查）。  
- 导出成功率与字段完整性。

## 6. 非目标

- 行业级剧本协同审批流（本阶段）。  
- 自动评价剧本艺术质量。

## 7. 风险

- 快照过多导致噪声 → 需合并策略与摘要视图（后续）。
