# Branch Tree · 《雾灯残页》游戏结构示意图 — 架构概览

> 产品定位：用羊皮纸手稿风格可视化展示文字冒险游戏的完整叙事架构——世界观/角色/分支轴线/章节时序/结局谱系。

---

## 1. 模块组成

```
┌───────────────────────────────────────────────────┐
│         Branch Tree · 游戏结构示意图                 │
├───────────┬───────────┬───────────┬──────────────┤
│  Board    │  Legend   │ Timeline  │  Ending Grid  │
│  主展板    │  图例面板  │  时间线    │  结局谱系      │
├───────────┴───────────┴───────────┴──────────────┤
│           Card 系统（羊皮纸做旧质感）                │
├───────────────────────────────────────────────────┤
│  响应式策略：mobile accordion ↔ desktop multi-col  │
└───────────────────────────────────────────────────┘
```

| 模块 | 职责 | 关键交互 |
|------|------|----------|
| **Board** | 全局展板容器，深褐底 + 径向渐变光晕 + drop shadow | 最大宽度 min(1320px, 96vw) |
| **Legend** | 图例面板：核心概念 + 三条轴线 + 结局类型标签 | auto-fit Grid，药丸 Chip |
| **Card 系统** | 统一的羊皮纸卡片：渐变 + 内边框 + drop shadow | 径向高光模拟光照效果 |
| **Timeline** | 章节时序：桌面 4 列 Grid，移动端 JS accordion 折叠 | 响应式断点切换 |
| **Ending Grid** | 结局谱系：接入条件 + 描述 + 达成条件 | Grid 自适应 |
| **Axis Chart** | 三条轴线可视化：命运轴/自我轴/未知轴 | 颜色编码区分 |

---

## 2. 数据流（静态展示）

```
页面加载 → CSS Grid 渲染 → 移动端 JS accordion 初始化
                              ↓
                        点击章节标题 → 展开/折叠
                              ↓
                        桌面端无 JS 交互，纯 CSS Grid 展示
```

**核心规则**：
- 纯静态展示页，无动态数据
- 移动端的 Timeline 通过 JS 实现 accordion（桌面端纯 Grid）
- 所有数据直接写入 HTML 结构，不经过 API 或存储

## 3. 核心样式系统

### 羊皮纸卡片

```css
.card {
  background:
    /* 顶部径向高光 — 模拟光照 */
    radial-gradient(ellipse at 50% 0%, rgba(255,240,210,0.6) 0%, transparent 60%),
    /* 垂直渐变 — 纸色过渡 */
    linear-gradient(180deg, #f5e9d4, var(--paper));
  background-color: #ead9bc;
  border: 1px solid #c3a87d;
  border-radius: 12px;
  box-shadow:
    inset 0 1px 0 rgba(255,255,255,0.55),  /* 顶部白边 — 纸张叠放 */
    0 4px 10px rgba(0,0,0,0.2);            /* 外部阴影 — 深度 */
}
```

### 三条轴线

| 轴线 | 色值 | 含义 | 视觉效果 |
|------|------|------|----------|
| 命运轴 | `#7d5b2a` 棕色 | 外部事件、不可抗力 | 稳定的地色 |
| 自我轴 | `#5e3e68` 紫色 | 角色成长、内在选择 | 精神性色调 |
| 未知轴 | `#2f5b70` 蓝色 | 隐藏线索、秘密结局 | 深邃探索感 |

## 4. 响应式策略

| 断点 | 布局规则 |
|------|----------|
| **Mobile (< 640px)** | 全单列（1fr），Timeline JS accordion，Legend 2 列 |
| **Tablet (640-1024px)** | Row 2 列，Lens Preview 2 列 |
| **Desktop (> 1024px)** | Row 2 列，Timeline 4 列 Grid，Lens Preview 3 列，Ending Grid 2 列 |

---

## 5. 设计决策

| 决策 | 理由 | 约束 |
|------|------|------|
| 羊皮纸手稿风格 | 与游戏哥特暗色叙事一致，形成「考古学家展板」隐喻 | 做旧效果依赖多层 CSS 渐变，不建议改色系 |
| 纯静态展示 | 信息架构图不需要交互，降低维护成本 | 无法动态联动游戏数据 |
| 移动端 accordion | Timeline 在移动端无法 4 列展示，折叠是最小损失方案 | 桌面端需写两套样式 |
| Georgia 衬线字体 | 免费系统字体，羊皮纸/古文献感强 | 中文字体回退到 Microsoft YaHei |

## 6. 与其他项目的关系

- **上游依赖**：数据来源于 `vn-game.html` 和 AIGC 生成资产（story_overview.md, worldbuilding.md）
- **设计系统**：色彩体系与 mistlamp.html 共享哥特暗色基底，但卡片色更偏向羊皮纸浅色
- **可迁移性**：卡片 + Chip + Legend 三件套可复用于任何「信息架构可视化」场景
