# 住值·租房生命成本 — 架构概览

> 本文档用最简语言回答三个问题：由哪些大块组成、大块之间怎么通信、核心数据长什么样。

---

## 1. 模块组成

```
┌─────────────────────────────────────────────────────┐
│                  住值·租房生命成本                    │
├─────────────┬─────────────┬─────────────┬───────────┤
│  租房记录   │  成本计算   │  决策建议   │  数据看板 │
│ (Rental     │ (Cost       │ (Decision   │ (Dashboard│
│  Record)    │  Calculator)│  Advisor)   │  )        │
├─────────────┴─────────────┴─────────────┴───────────┤
│                   数据层 (localStorage)               │
├─────────────────────────────────────────────────────┤
│  成本模型  │  可视化 (Chart.js)  │  导出            │
└─────────────────────────────────────────────────────┘
```

| 模块 | 职责 | 关键交互 |
|------|------|----------|
| **租房记录** | 录入租房信息（租金/押金/水电/交通） | 为成本计算提供数据 |
| **成本计算** | 计算真实租房成本（含时间/交通/生活成本） | 从租房记录读取；供决策建议使用 |
| **决策建议** | 基于成本分析给出租房建议 | 从成本计算读取；供用户参考 |
| **数据看板** | 成本可视化、对比分析 | 读取租房记录，聚合展示 |
| **成本模型** | 多维度成本计算模型 | 驱动成本计算模块 |
| **导出** | JSON 备份、报告导出 | 读取全量数据，格式化后触发下载 |

## 2. 数据流

```
用户操作                 数据流向                     存储
─────────────────────────────────────────────────────────
录入租房 ──────────────→ Rental 对象写入 ──────────→ rentals[]
                                                  ↓
计算成本 ←───────────── Cost 计算 ──────────────→ costs[]
                                                  ↓
生成建议 ←───────────── Advice 生成 ──────────────→ advices[]
                                                  ↓
可视化 ←───────────── Chart.js 渲染 ──────────→ dashboard
```

**核心规则**：
- 所有数据默认写 `localStorage`
- 成本模型包含：租金+押金+水电+交通+时间成本
- 决策建议基于成本对比分析

## 3. 核心数据结构

### Rental（租房记录）

```javascript
{
  id: "rental_1694000000",
  address: "XX小区",
  rent: 3000,           // 月租
  deposit: 6000,        // 押金
  utilities: 500,       // 水电
  commuteTime: 60,      // 通勤时间（分钟）
  commuteCost: 300,     // 交通费
  leaseStart: "2025-01-01",
  leaseEnd: "2025-12-31",
  tags: ["近地铁", "独立卫浴"],
  createdAt: "2025-01-01T08:00:00Z"
}
```

### Cost（成本计算）

```javascript
{
  id: "cost_1694000001",
  rentalId: "rental_1694000000",
  monthlyCost: 3800,      // 月总成本
  yearlyCost: 45600,      // 年总成本
  timeCost: 15600,        // 年时间成本（按小时工资计算）
  totalCost: 61200,       // 年总成本（含时间）
  createdAt: "2025-01-01T08:00:00Z"
}
```

## 4. 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 存储方案 | localStorage | 零服务器成本 |
| 成本模型 | 多维度（租金+时间+交通） | 访谈中用户「只看租金，忽略隐性成本」 |
| 可视化 | Chart.js | 直观展示成本构成 |
| 框架选择 | 纯 HTML/JS | 零构建成本，GitHub Pages 直接托管 |
