# Prompt Library — 架构概览

> 本文档用最简语言回答三个问题：由哪些大块组成、大块之间怎么通信、核心数据长什么样。

---

## 1. 模块组成

```
┌─────────────────────────────────────────────────────┐
│                   Prompt Library                    │
├─────────────┬─────────────┬─────────────┬───────────┤
│  Prompt 管理 │  分类系统   │  一键复制   │  搜索过滤 │
│ (Prompt     │ (Category   │ (Copy       │ (Search   │
│  Manager)   │  System)    │  Function)  │  Filter)  │
├─────────────┴─────────────┴─────────────┴───────────┤
│                   数据层 (localStorage)               │
├─────────────────────────────────────────────────────┤
│  导入/导出  │  标签系统  │  使用统计                   │
└─────────────────────────────────────────────────────┘
```

| 模块 | 职责 | 关键交互 |
|------|------|----------|
| **Prompt 管理** | 创建/编辑/删除 Prompt | 为分类系统和一键复制提供数据 |
| **分类系统** | 多级分类管理 | 供 Prompt 管理和搜索过滤使用 |
| **一键复制** | 快速复制 Prompt 到剪贴板 | 从 Prompt 管理读取 |
| **搜索过滤** | 按关键词/分类/标签过滤 | 读取 Prompt 管理，聚合展示 |
| **导入/导出** | JSON 导入/导出 | 读取全量数据，格式化后触发下载 |
| **标签系统** | 标签管理，支持高频标签推荐 | 供 Prompt 管理使用 |

## 2. 数据流

```
用户操作                 数据流向                     存储
─────────────────────────────────────────────────────────
创建 Prompt ──────────→ Prompt 对象写入 ──────────→ prompts[]
                                                  ↓
分类管理 ──────────────→ Category 更新 ──────────→ categories[]
                                                  ↓
搜索过滤 ←───────────── 过滤查询 ──────────────→ 按关键词/分类聚合
                                                  ↓
一键复制 ──────────────→ 剪贴板写入 ──────────────→ 浏览器 API
```

**核心规则**：
- 所有数据默认写 `localStorage`
- 支持多级分类（如：写作/营销/编程）
- 一键复制使用浏览器 Clipboard API

## 3. 核心数据结构

### Prompt（提示词）

```javascript
{
  id: "prompt_1694000000",
  title: "文章大纲生成",
  content: "请帮我生成一篇关于{topic}的文章大纲...",
  category: "写作",
  tags: ["大纲", "写作"],
  usageCount: 15,
  createdAt: "2025-12-25T08:00:00Z",
  updatedAt: "2025-12-25T08:00:00Z"
}
```

### Category（分类）

```javascript
{
  id: "cat_1694000001",
  name: "写作",
  parentId: null,        // 支持多级分类
  promptCount: 12,
  createdAt: "2025-12-25T08:00:00Z"
}
```

---

## 4. 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 存储方案 | localStorage | 零服务器成本，隐私优先 |
| 分类方案 | 多级分类 + 标签 | 灵活组织 Prompt |
| 复制方案 | Clipboard API | 一键复制，提升效率 |
| 框架选择 | 纯 HTML/JS | 零构建成本，GitHub Pages 直接托管 |
