# 会议·反刍舱 — 架构概览

> 本文档用最简语言回答三个问题：由哪些大块组成、大块之间怎么通信、核心数据长什么样。

---

## 1. 模块组成

```
┌─────────────────────────────────────────────────────┐
│                    会议·反刍舱                       │
├─────────────┬─────────────┬─────────────┬───────────┤
│  会议记录   │  待办台账   │  复盘笔记   │  状态流   │
│ (Meeting    │ (Todo       │ (Review     │ (Status   │
│  Record)    │  Ledger)    │  Notes)     │  Flow)    │
├─────────────┴─────────────┴─────────────┴───────────┤
│                   数据层 (localStorage)               │
├─────────────────────────────────────────────────────┤
│  语音转写 (Whisper)  │  导出 (JSON/Excel/Word)      │
└─────────────────────────────────────────────────────┘
```

| 模块 | 职责 | 关键交互 |
|------|------|----------|
| **会议记录** | 录入/转写会议内容，结构化提取待办 | 写入待办台账；供复盘模块关联 |
| **待办台账** | 管理待办事项，支持状态流转（待办→进行中→完成） | 从会议记录读取；供复盘模块统计 |
| **复盘笔记** | 记录会议复盘，支持标签筛选 | 关联待办台账，支持按会议/按标签筛选 |
| **状态流** | 待办状态流转（待办→进行中→完成→归档） | 驱动待办台账的状态变更 |
| **语音转写** | 本地 Whisper 转写会议录音 | 结果直入会议记录，用户确认后落库 |
| **导出** | JSON 备份、Excel 导出、Word 导出 | 读取全量数据，格式化后触发下载 |

## 2. 数据流

```
用户操作                 数据流向                     存储
─────────────────────────────────────────────────────────
录入会议 ──────────────→ Meeting 对象写入 ──────────→ meetings[]
                                                  ↓
提取待办 ←───────────── Todo 对象写入 ──────────────→ todos[]
                                                  ↓
状态流转 ←───────────── Status 更新 ──────────────→ todos[].status
                                                  ↓
复盘记录 ←───────────── Review 对象写入 ──────────→ reviews[]
                                                  ↓
导出报告 ←───────────── 格式化输出 ──────────────→ 下载
```

**核心规则**：
- 所有数据默认写 `localStorage`，不经过任何云端
- 待办状态机：待办 → 进行中 → 完成 → 归档
- 语音转写仅作为内容预填，用户确认后才落库

## 3. 核心数据结构

### Meeting（会议记录）

```javascript
{
  id: "meet_1694000000",
  title: "产品周会",
  date: "2025-11-15",
  participants: ["张三", "李四"],
  content: "会议内容...",
  todos: ["todo_001", "todo_002"],
  tags: ["产品", "周会"],
  createdAt: "2025-11-15T10:00:00Z"
}
```

### Todo（待办事项）

```javascript
{
  id: "todo_1694000001",
  meetingId: "meet_1694000000",
  title: "完成PRD评审",
  assignee: "张三",
  status: "todo",          // todo | in_progress | done | archived
  priority: "high",        // high | medium | low
  dueDate: "2025-11-20",
  createdAt: "2025-11-15T10:00:00Z"
}
```

### Review（复盘笔记）

```javascript
{
  id: "rev_1694000002",
  meetingId: "meet_1694000000",
  content: "本次会议复盘...",
  tags: ["复盘", "产品"],
  createdAt: "2025-11-15T11:00:00Z"
}
```

## 4. 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 存储方案 | localStorage | 隐私优先，零服务器成本 |
| 转写方案 | Whisper（本地） | 访谈中用户「不想上传公网」 |
| 状态机 | 待办→进行中→完成→归档 | 简洁明了，覆盖会议待办全生命周期 |
| 框架选择 | 纯 HTML/JS | 零构建成本，GitHub Pages 直接托管 |
