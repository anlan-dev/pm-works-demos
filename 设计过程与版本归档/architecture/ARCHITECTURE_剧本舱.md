# 剧本·复盘舱 — 架构概览

> 本文档用最简语言回答三个问题：由哪些大块组成、大块之间怎么通信、核心数据长什么样。

---

## 1. 模块组成

```
┌─────────────────────────────────────────────────────┐
│                   剧本·复盘舱                       │
├─────────────┬─────────────┬─────────────┬───────────┤
│  剧本管理   │  进度快照   │  复盘笔记   │  标签系统 │
│ (Script     │ (Progress   │ (Review     │ (Tag      │
│  Manager)   │  Snapshot)  │  Notes)     │  System)  │
├─────────────┴─────────────┴─────────────┴───────────┤
│                   数据层 (localStorage)               │
├─────────────────────────────────────────────────────┤
│  导出 (Word/CSV/JSON)  │  热力图 (Streak)           │
└─────────────────────────────────────────────────────┘
```

| 模块 | 职责 | 关键交互 |
|------|------|----------|
| **剧本管理** | 分场管理剧本，支持多剧本并行 | 为进度快照和复盘笔记提供上下文 |
| **进度快照** | 每次修改自动记录快照 | 写入剧本管理；供复盘模块回溯 |
| **复盘笔记** | 记录修改原因，支持标签筛选 | 关联进度快照，支持按标签/按场次筛选 |
| **AI 剧情引擎** | 基于剧本设定生成剧情走向、角色对白、场景描述 | 调用 LLM API；结果预填到剧本编辑器 |
| **标签系统** | 标签管理，支持高频标签推荐 | 供复盘笔记和进度快照使用 |
| **导出** | Word/CSV/JSON 导出 | 读取全量数据，格式化后触发下载 |
| **热力图** | 写作 streak 可视化 | 读取写作记录，聚合展示 |

## 2. 数据流

```
用户操作                 数据流向                     存储
─────────────────────────────────────────────────────────
创建剧本 ──────────────→ Script 对象写入 ──────────→ scripts[]
                                                  ↓
修改剧本 ──────────────→ Snapshot 自动记录 ──────→ scripts[].snapshots[]
                                                  ↓
填写复盘 ──────────────→ Review 对象写入 ──────────→ reviews[]
                                                  ↓
标签筛选 ←───────────── 过滤查询 ──────────────→ 按标签聚合
```

**核心规则**：
- 所有数据默认写 `localStorage`
- 每次修改自动记录进度快照
- 复盘笔记支持标签筛选

## 3. 核心数据结构

### Script（剧本）

```javascript
{
  id: "script_1694000000",
  title: "我的剧本",
  genre: "悬疑",
  scenes: [
    { id: "scene_001", title: "第一场", content: "..." }
  ],
  snapshots: [
    {
      id: "snap_001",
      date: "2025-12-20T10:00:00Z",
      changeType: "delete_scene",
      changeDetail: "删除第三场",
      reason: "节奏太慢"
    }
  ],
  tags: ["悬疑", "长篇"],
  createdAt: "2025-12-20T08:00:00Z"
}
```

### Review（复盘笔记）

```javascript
{
  id: "rev_1694000001",
  scriptId: "script_1694000000",
  snapshotId: "snap_001",
  content: "删除第三场是因为节奏太慢...",
  tags: ["删戏", "节奏"],
  createdAt: "2025-12-20T10:00:00Z"
}
```

## 4. 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 存储方案 | localStorage | 零服务器成本 |
| 快照策略 | 每次修改自动记录 | 访谈中用户「忘了某场戏为什么删」 |
| 标签系统 | 高频标签推荐 + 历史统计 | 防止标签滥用 |
| 框架选择 | 纯 HTML/JS | 零构建成本，GitHub Pages 直接托管 |
