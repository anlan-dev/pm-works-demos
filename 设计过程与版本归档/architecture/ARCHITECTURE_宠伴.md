# 宠伴·健康台账 — 架构概览

> 本文档用最简语言回答三个问题：由哪些大块组成、大块之间怎么通信、核心数据长什么样。

---

## 1. 模块组成

```
┌─────────────────────────────────────────────────────┐
│                    宠伴·健康台账                       │
├─────────────┬─────────────┬─────────────┬───────────┤
│  宠物档案   │  疫苗日历   │  花费看板   │  智能提醒  │
│  (Pet CRUD) │ (Vaccine    │ (Expense    │ (Reminder │
│             │  Calendar)  │  Dashboard) │  Engine)  │
├─────────────┴─────────────┴─────────────┴───────────┤
│                   数据层 (localStorage)               │
├─────────────────────────────────────────────────────┤
│  OCR 引擎 (Tesseract.js)  │  导出 (SheetJS/Chart.js) │
└─────────────────────────────────────────────────────┘
```

| 模块 | 职责 | 关键交互 |
|------|------|----------|
| **宠物档案** | 创建/编辑/删除宠物信息（名称、品种、头像、生日） | 为疫苗、花费、提醒提供归属上下文 |
| **疫苗日历** | 按宠物录入疫苗/驱虫/体检记录，自动推算下次接种日 | 写入提醒队列；供花费模块关联医疗支出 |
| **花费看板** | 记录医疗/食品/用品支出，按月聚合，Chart.js 可视化 | 关联宠物档案，支持按宠筛选 |
| **智能提醒** | 按宠物独立提醒队列，支持「稍后」「今日忽略」 | 从疫苗日历读取计划日，计算提醒时机 |
| **OCR 引擎** | 本地识别接种凭证照片，提取疫苗名称/日期 | 结果直入疫苗日历，用户确认后落库 |
| **导出** | JSON 备份、Excel 导出、周报自动生成 | 读取全量数据，格式化后触发下载 |

## 2. 数据流

```
用户操作                 数据流向                     存储
─────────────────────────────────────────────────────────
添加宠物 ──────────────→ Pet 对象写入 localStorage ──→ pets[]
录入疫苗 ──────────────→ Vaccine 对象写入 ──────────→ vaccines[]
                                                  ↓
                                           推算 nextDate
                                                  ↓
设置提醒 ←───────────── Reminder 对象写入 ──────────→ reminders[]
                                                  ↓
提醒触发 ←───────────── 定时检查 nextDate ─────────→ 弹窗/通知
OCR 拍照 → Tesseract.js → 预填表单 → 用户确认 → vaccines[]
```

**核心规则**：
- 所有数据默认写 `localStorage`，不经过任何云端
- 提醒引擎定时扫描 `reminders[]`，与当前日期比较后触发
- OCR 结果仅作为表单预填，用户确认后才落库

## 3. 核心数据结构

### Pet（宠物档案）

```javascript
{
  id: "pet_1694000000",      // 唯一标识
  name: "豆豆",              // 宠物名称
  species: "犬",             // 物种
  breed: "柯基",             // 品种
  birthday: "2022-05-01",    // 生日（可选）
  avatar: "data:image/...",  // Base64 头像（可选）
  createdAt: "2025-09-15T08:00:00Z"
}
```

### Vaccine（疫苗/健康记录）

```javascript
{
  id: "vac_1694000001",
  petId: "pet_1694000000",   // 关联宠物
  type: "vaccine",           // 类型：vaccine | deworm | checkup
  name: "犬五联",            // 疫苗名称
  date: "2025-09-15",        // 接种日期
  nextDate: "2026-09-15",    // 自动推算的下次接种日
  batchNo: "H20250801",      // 批号（可选）
  hospital: "XX宠物医院",    // 医院（可选）
  note: "",                  // 备注
  createdAt: "2025-09-15T08:00:00Z"
}
```

### Reminder（提醒对象）

```javascript
{
  id: "rem_1694000002",
  petId: "pet_1694000000",   // 关联宠物
  vaccineId: "vac_1694000001", // 关联疫苗
  scheduledDate: "2026-09-01", // 计划提醒日
  status: "pending",          // pending | snoozed | dismissed | done
  snoozeUntil: null,          // 稍后提醒的目标时间
  createdAt: "2025-09-15T08:00:00Z"
}
```

### Expense（花费记录）

```javascript
{
  id: "exp_1694000003",
  petId: "pet_1694000000",   // 关联宠物
  category: "medical",       // medical | food | supplies | other
  amount: 320,               // 金额（元）
  date: "2025-09-15",
  note: "犬五联疫苗",
  createdAt: "2025-09-15T08:00:00Z"
}
```

## 4. 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 存储方案 | localStorage | 访谈中 83% 用户拒绝云同步；零服务器成本 |
| OCR 方案 | Tesseract.js（本地） | 隐私优先，无网络依赖 |
| 提醒策略 | 按宠物独立 + 稍后/忽略 | A/B 测试显示配置意愿提升 |
| 框架选择 | 纯 HTML/JS | 零构建成本，GitHub Pages 直接托管 |
