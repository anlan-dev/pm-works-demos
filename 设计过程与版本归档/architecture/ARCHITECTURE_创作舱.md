# 星途·创作舱 — 架构概览

> 本文档用最简语言回答三个问题：由哪些大块组成、大块之间怎么通信、核心数据长什么样。

---

## 1. 模块组成

```
┌─────────────────────────────────────────────────────┐
│                    星途·创作舱                       │
├─────────────┬─────────────┬─────────────┬───────────┤
│  作品管理   │  写作桌面   │  灵感工具   │  数据看板 │
│ (Works      │ (Writing    │ (Inspiration│ (Dashboard│
│  Manager)   │  Desk)      │  Tools)     │  )        │
├─────────────┴─────────────┴─────────────┴───────────┤
│                   数据层 (localStorage)               │
├─────────────────────────────────────────────────────┤
│  自动快照  │  多主题切换  │  导出 (JSON)              │
└─────────────────────────────────────────────────────┘
```

| 模块 | 职责 | 关键交互 |
|------|------|----------|
| **作品管理** | 创建/管理多部作品，支持多书并行 | 为写作桌面提供作品上下文 |
| **写作桌面** | 沉浸式写作界面，支持多种主题 | 写入作品管理；供灵感工具关联 |
| **灵感工具** | 标题灵感库、随机抽题、字数目标 | 从作品管理读取；供写作桌面使用 |
| **AI 灵感引擎** | 基于关键词生成标题、开头段落、角色设定 | 调用 LLM API；结果预填到写作桌面 |
| **数据看板** | 字数统计、写作 streak、进度可视化 | 读取写作记录，聚合展示 |
| **自动快照** | 定时自动保存，防崩溃丢失 | 写入 localStorage |
| **多主题切换** | 默认/暗色/墨水屏/护眼 | 切换 CSS 变量 |

## 2. 数据流

```
用户操作                 数据流向                     存储
─────────────────────────────────────────────────────────
创建作品 ──────────────→ Work 对象写入 ──────────────→ works[]
                                                  ↓
开始写作 ──────────────→ Content 更新 ──────────────→ works[].content
                                                  ↓
自动快照 ──────────────→ 定时保存 ──────────────→ localStorage
                                                  ↓
字数统计 ←───────────── 聚合计算 ──────────────→ dashboard
```

**核心规则**：
- 所有数据默认写 `localStorage`，无账号体系
- 自动快照每 30 秒触发一次
- 多主题切换通过 CSS 变量实现

## 4. 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 存储方案 | localStorage | 无账号体系，零服务器成本 |
| 自动保存 | 30 秒快照 | 访谈中用户反馈「写到一半崩溃丢稿」 |
| 主题方案 | CSS 变量 | 支持默认/暗色/墨水屏/护眼 |
| 框架选择 | 纯 HTML/JS | 零构建成本，GitHub Pages 直接托管 |
