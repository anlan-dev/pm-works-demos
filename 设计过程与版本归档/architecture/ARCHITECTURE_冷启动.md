# Cold Start · 冷启动 — 架构概览

> 本文档用最简语言回答三个问题：由哪些大块组成、大块之间怎么通信、核心数据长什么样。

---

## 1. 模块组成

```
┌─────────────────────────────────────────────────────┐
│                 Cold Start · 冷启动                   │
├─────────────┬─────────────┬─────────────┬───────────┤
│  拖延诊断   │  启动任务   │  后续计划   │  AI 引擎  │
│ (Diagnosis  │ (Starter    │ (Follow-up  │ (AI       │
│  )          │  Tasks)     │  Plan)      │  Engine)  │
├─────────────┴─────────────┴─────────────┴───────────┤
│                   数据层 (localStorage)               │
├─────────────────────────────────────────────────────┤
│  规则引擎（离线）  │  AI API（多平台）               │
└─────────────────────────────────────────────────────┘
```

| 模块 | 职责 | 关键交互 |
|------|------|----------|
| **拖延诊断** | 分析拖延原因，生成诊断报告 | 为启动任务和后续计划提供依据 |
| **启动任务** | 生成 5 个极小启动任务 | 从诊断读取；供用户执行 |
| **后续计划** | 生成后续行动计划 | 从诊断读取；供用户参考 |
| **AI 引擎** | 调用 AI API 生成诊断/任务/计划 | 支持 OpenRouter/DeepSeek/SiliconFlow |
| **规则引擎** | 离线兜底，AI 失败时自动降级 | 确保基本功能可用 |

## 2. 数据流

```
用户操作                 数据流向                     存储
─────────────────────────────────────────────────────────
输入目标 ──────────────→ Goal 对象写入 ──────────────→ goals[]
                                                  ↓
AI 诊断 ──────────────→ Diagnosis 生成 ──────────→ goals[].diagnosis
                                                  ↓
生成任务 ←───────────── Tasks 生成 ──────────────→ goals[].tasks
                                                  ↓
执行任务 ──────────────→ Task 状态更新 ──────────→ goals[].tasks[].done
                                                  ↓
后续计划 ←───────────── Plan 生成 ──────────────→ goals[].plan
```

**核心规则**：
- 所有数据默认写 `localStorage`
- AI 调用支持多平台，可选离线规则引擎
- 任务粒度极小（5 分钟内可完成）

## 3. 核心数据结构

### Goal（拖延目标）

```javascript
{
  id: "goal_1694000000",
  title: "完成毕业论文",
  description: "拖延了两个月...",
  diagnosis: {
    reason: "恐惧失败",
    type: "perfectionism",
    confidence: 0.85
  },
  tasks: [
    { id: "task_001", title: "打开文档，写下标题", done: false },
    { id: "task_002", title: "列出三个大纲要点", done: false }
  ],
  plan: {
    summary: "分三个阶段完成",
    steps: ["第一周：大纲", "第二周：初稿", "第三周：修改"]
  },
  createdAt: "2025-12-20T08:00:00Z"
}
```

### AIConfig（AI 配置）

```javascript
{
  provider: "openrouter",    // openrouter | deepseek | siliconflow
  apiKey: "sk-xxx",
  model: "gpt-4",
  useOfflineFallback: true   // AI 失败时自动降级到规则引擎
}
```

## 4. 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 存储方案 | localStorage | 零服务器成本，隐私优先 |
| AI 方案 | 多平台支持 + 离线兜底 | 确保基本功能可用 |
| 任务粒度 | 极小（5 分钟内） | 访谈中用户「怕开始」 |
| 框架选择 | 纯 HTML/JS | 零构建成本，PWA 就绪 |
