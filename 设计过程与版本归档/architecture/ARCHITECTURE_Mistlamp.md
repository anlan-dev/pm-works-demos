# 雾灯残页 · 架构概览（游戏本体 + AIGC 展示页）

> 产品定位：全程 AIGC 辅助开发的哥特暗色文字冒险游戏（Interactive Fiction），含可玩本体（mistlamp.html）和能力展示页（vn-game.html）+ 结构示意图（branch_tree.html）。

---

## 1. 模块组成

```
┌─────────────────────────────────────────────────────────┐
│                雾灯残页 · 项目体系                         │
├───────────────────┬───────────────────┬─────────────────┤
│  mistlamp.html    │  vn-game.html     │ branch_tree.html │
│  游戏本体          │  AIGC 展示页       │  结构示意图       │
├───────────────────┴───────────────────┴─────────────────┤
│         共用色彩体系（哥特暗色 + 羊皮纸浅色）               │
```

### mistlamp.html — 游戏本体

| 模块 | 职责 | 关键交互 |
|------|------|----------|
| **主菜单** | 新游戏/继续/存档管理入口 | 3 个按钮，径向渐变背景 |
| **属性系统** | 管理玩家属性（意志/洞察/共鸣/暗蚀），影响分支可用性 | 属性条实时展示，选择影响属性变化 |
| **叙事引擎** | 显示章节文本，根据属性值筛选可用分支 | Noto Serif SC 衬线正文，1.8 行高 |
| **分支选择** | 展示可选行动，每项可能有属性门槛 | 全宽按钮，≥48px 触控区域 |
| **存档系统** | localStorage 保存/读取/删除存档 | 多个存档槽位 |
| **设置面板** | 主题切换（默认暗色/更深/羊皮纸浅色） | 全局 CSS 变量重载 |

### vn-game.html — AIGC 展示页

| 模块 | 职责 | 关键交互 |
|------|------|----------|
| **Hero** | 项目概述 + 核心数据 | Stats Grid（万字/AI调用/结局数） |
| **过程展示** | 按阶段展示 AIGC 应用：世界观→角色→分支→润色→导出→翻译 | 折叠 Details 面板 |
| **文件溯源** | 直接链接生成资产文件 | file-link 组件 |
| **技术架构** | AI 模型选择 + Prompt 策略 + 质量控制 | 引用块 + 故事块 |

### branch_tree.html — 结构示意图

| 模块 | 职责 | 关键交互 |
|------|------|----------|
| **Board 展板** | 羊皮纸风格的全局容器 | 深底 + 做旧卡片 |
| **Legend 图例** | 三条轴线 + 结局类型标注 | 药丸 Chip 标签 |
| **Timeline** | 章节时序展示（移动端 accordion） | 桌面 4 列，移动端折叠 |
| **Ending Grid** | 结局谱系（接入条件 + 描述） | Grid 自适应 |

## 2. 数据流（游戏本体）

```
用户操作                    数据流向                      存储
──────────────────────────────────────────────────────────
新游戏 ──────────────→ 初始化属性 + 章节指针 ────→ gameState
                                                        ↓
阅读文本 ────────────→ 展示叙事段落 ──────────────→ gameState.chapter
                                                        ↓
选择分支 ────────────→ 检查属性门槛 → 应用属性变动 ──→ gameState.stats
                                                        ↓
推进章节 ────────────→ 下一段叙事 ──────────────────→ gameState
                                                        ↓
保存 ────────────────→ JSON序列化 ──────────────────→ localStorage
                                                        ↓
读档 ────────────────→ JSON反序列化 ─────────────────→ gameState 恢复
```

**核心规则**：
- 分支选择基于属性门槛（如「意志≥5 才能选择正面冲突」）
- 选择后属性实时变动，影响后续可用分支
- 存档全量保存 gameState，包含属性/章节/选择历史

## 3. 核心数据结构

### GameState（游戏本体）

```javascript
{
  chapter: 1,              // 当前章节
  scene: 3,                // 当前场景
  stats: {
    will: 5,               // 意志
    insight: 3,            // 洞察
    resonance: 4,          // 共鸣
    corruption: 2          // 暗蚀
  },
  history: [               // 选择历史（用于回滚/结局判定）
    { chapter: 1, scene: 1, choice: 2 },
    { chapter: 1, scene: 2, choice: 0 }
  ],
  flags: {                 // 剧情标记（影响分支可见性）
    met_elder: true,
    found_lamp: false
  },
  startedAt: 'ISO日期',
  updatedAt: 'ISO日期'
}
```

## 4. 设计决策

| 决策 | 理由 | 约束 |
|------|------|------|
| 三文件分离（游戏/展示/结构图） | 面向不同受众：玩家/面试官/协作者 | 需维护三套 HTML 但共享色彩体系 |
| 衬线叙事字体 | 增强「手稿/古书」阅读沉浸感 | 屏幕阅读性弱于无衬线，需加大行高（1.8） |
| 属性门槛分支系统 | 传统 IF 经典设计，降低开发复杂度同时保持可玩性 | 需在设计阶段规划所有门槛组合 |
| localStorage 存档 | Demo 阶段零后端成本，离线可用 | 清浏览器缓存会丢失，需在产品说明中标明 |
| 三主题切换 | 兼容不同阅读场景（夜间/白天/打印） | 需为每种主题独立维护 CSS 变量 |
| AIGC 全程开发 | 展示「PM 如何指导 AI 完成复杂创意项目」的核心能力 | AI 生成内容的版权和质量需声明，见 vn-game.html |

## 6. 与其他项目的关系

- **设计系统**：暗色体系独立于工具效率类白卡片风格，属于消费娱乐类沉浸暗色
- **AIGC 能力**：本项目的核心展示价值——全程 AI 辅助开发的完整案例
- **分支树**：可复用的叙事架构可视化方法，未来可迁移到其他互动叙事项目
